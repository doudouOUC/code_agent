## 解决的问题

subagent 使用 approval-mode override 继承父级 plan mode 后，`exit_plan_mode` 即使成功也仍会从固定 getter 读回初始 plan mode，导致子 agent 无法真正进入执行态。AUTO 模式下的 dangerous-rule denial cleanup 也依赖初始 mode，子 config 在运行中进入或离开 AUTO 时可能不能按实际状态恢复共享 PermissionManager。

## 最终实现方案

`packages/core/src/tools/agent/agent.ts` 的 `createApprovalModeOverride` 改为持有可变 approval mode 和 plan gate state：子 config 的 `getApprovalMode` 读取当前 override state，`setApprovalMode` / plan gate 更新同一份状态，因此 `exit_plan_mode` 成功后后续工具权限判断能看到新 mode。

AUTO dangerous-rule 处理改为 child-local tracking：子 agent 启动、进入 AUTO、离开 AUTO 或结束时按当前状态安装/恢复 denial rules，不再只看初始 approval mode。`packages/core/src/tools/agent/agent-override.test.ts` 覆盖 subagent 退出 plan mode、AUTO cleanup 和共享 permission manager 恢复边界。

## 关键代码路径

- `packages/core/src/tools/agent/agent.ts`: `createApprovalModeOverride` 改为可变 approval mode/plan gate state，并隔离 child-local AUTO dangerous-rule denial tracking。
- `packages/core/src/tools/agent/agent-override.test.ts`: 覆盖 subagent 退出 plan mode、AUTO cleanup、共享 PermissionManager 恢复。

## 验证方式

- `cd packages/core && npx vitest run src/tools/agent/agent-override.test.ts`: 验证子 agent `exit_plan_mode` 后权限状态真实变化，并验证 AUTO denial rules 生命周期。
