## 解决的问题

open：daemon 用户需要把旧 session 从 active 列表中移走，但又不能删除 transcript 和项目元数据。当前没有明确的 archive/unarchive 语义，也没有跨 REST、ACP vendor method、HTTP-over-ACP 和 SDK 的统一入口。

## 当前实现观察

PR 分支计划用 session JSONL 的物理位置表达 active 与 archived 状态：active session 位于 `chats/`，archived session 移到 `chats/archive/`。daemon 可列出 active/archived session、archive inactive 或 live session、拒绝 archived load/resume、unarchive 回 active storage，并删除 archived JSONL。

主要路径包括 `packages/cli/src/serve/server/session-archive.ts`、`routes/session.ts`、`server/session-list.ts`、`acp-http/dispatch.ts`、`packages/acp-bridge/src/bridge.ts` / `bridgeErrors.ts`、`packages/core/src/services/sessionService.ts`，以及 `packages/sdk-typescript/src/daemon/DaemonClient.ts`、`acpRouteTable.ts`、`types.ts`。能力发现新增 session archive capability，协议文档和 daemon lifecycle 文档同步更新。
