## 解决的问题

daemon 用户需要把旧 session 从 active 列表中移走，但又不能删除 transcript、worktree sidecar 和项目元数据。原有 session list/delete/load/resume 只有 active 语义，没有 archive/unarchive 状态，也没有 REST、ACP vendor method、HTTP-over-ACP 和 TS SDK 的统一入口。

## 最终实现

PR 用 session JSONL 的物理位置表达状态：active session 位于 `chats/<id>.jsonl`，archived session 位于 `chats/archive/<id>.jsonl`。`packages/core/src/services/sessionService.ts` 增加 `getSessionLocation`、active/archived list、`archiveSessions`、`unarchiveSessions` 和 archived sidecar 清理，active/archive 双写会被视为 conflict。

serve 层新增 `packages/cli/src/serve/server/session-archive.ts`，提供 `archiveDaemonSessions`、`unarchiveDaemonSessions`、`deleteDaemonSessions` 和 `SessionArchiveCoordinator`。archive live session 时先 strict close 并要求 ACP close handler flush recording，成功后才移动 JSONL；load/resume/prompt/close/delete 与 archive/unarchive 之间通过 shared/exclusive gate 避免同一 id 被边读边移动。

协议面新增 `POST /sessions/archive`、`POST /sessions/unarchive`、`GET /workspace/:id/sessions?archiveState=active|archived`，并通过 `packages/cli/src/serve/acp-http/dispatch.ts` 映射到 `_qwen/sessions/archive` / `_qwen/sessions/unarchive`。`SessionArchivedError`、`SessionConflictError`、`SessionArchivingError` 被映射为 `409 session_archived` / `session_conflict` / `session_archiving`，archived session load/resume 会明确要求先 unarchive。

TS SDK 在 `packages/sdk-typescript/src/daemon/DaemonClient.ts`、`types.ts` 和 `acpRouteTable.ts` 增加 archiveState list、archive/unarchive client helpers 与 HTTP-over-ACP 路由。
