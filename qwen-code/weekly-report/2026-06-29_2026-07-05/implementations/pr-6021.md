## 解决的问题

ACP-backed `read_file` 在 `qwen serve` 下会先经过 workspace-bound filesystem。skill 指令、临时输出、subagent transcript、managed auto-memory、user extension 等路径本来是本地工具允许读取的 managed local roots，但在 serve 里会被 workspace boundary 误判为越界；同时 plain object 错误会被渲染成 `[object Object]`，让用户和 telemetry 都看不到真实原因。

## 最终实现方案

`packages/cli/src/acp-integration/service/filesystem.ts` 在 workspace read 被 boundary 拒绝后，只对 read-only 且命中受管本地根的路径 fallback 到 local filesystem service，仍然阻断非受管越界路径。`acpAgent.ts` 接入该行为，保持 ACP 工具调用与本地 `read_file` 的既有 allowlist 一致。

`packages/core/src/utils/errors.ts` 增加 plain object / structured JSON 的 message normalization，`read-file.ts`、`edit.ts`、`write-file.ts`、`fileUtils.ts` 统一调用，避免把结构化失败折叠成 `[object Object]`。覆盖包括 ACP filesystem、core tool、error rendering、fileUtils、serve fast-path bundle check 与 package script 测试。

## 关键代码路径

- `packages/cli/src/acp-integration/service/filesystem.ts`: workspace boundary 拒绝后，对 read-only managed local roots fallback 到 local filesystem。
- `packages/cli/src/acp-integration/acpAgent.ts`: ACP 工具路径接入 managed local read fallback。
- `packages/core/src/utils/errors.ts`: plain object / structured error message normalization。
- `packages/core/src/tools/read-file.ts`、`edit.ts`、`write-file.ts`、`utils/fileUtils.ts`: 工具错误渲染统一使用 normalized message。
- `scripts/check-serve-fast-path-bundle.js`、`scripts/tests/package-scripts.test.js`: 保持 serve fast path/package scripts 的静态检查不被新增导入破坏。

## 验证方式

- `cd packages/cli && npx vitest run src/acp-integration/service/filesystem.test.ts src/acp-integration/acpAgent.test.ts`: ACP managed local read fallback 与拒绝边界。

- `cd packages/core && npx vitest run src/tools/read-file.test.ts src/tools/edit.test.ts src/tools/write-file.test.ts`: read/edit/write 错误渲染回归。
