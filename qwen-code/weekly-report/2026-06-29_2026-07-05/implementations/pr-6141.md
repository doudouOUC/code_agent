## 解决的问题

core 工具生成 patch/diff 时默认 `ignoreWhitespace: true`，对普通内容改动能减少噪音，但纯缩进、空格或换行改动会产生 header-only patch。UI/工具结果因此显示 “No changes detected”，实际却已经修改了文件内容。

## 最终实现

`packages/core/src/tools/diffOptions.ts` 保留 `DEFAULT_DIFF_OPTIONS = { context: 3, ignoreWhitespace: true }`，新增 `hasHunks()`、`createPatchSmart()` 和内部 `structuredPatchSmart()`。策略是先用默认 ignoreWhitespace 生成干净 patch；如果 patch 没有 `@@` hunk，再以 `ignoreWhitespace:false` fallback 生成 whitespace-only diff。内容完全相同仍返回 no-hunk patch。

`edit.ts`、`notebook-edit.ts`、`write-file.ts`、`shell.ts`、`modifiable-tool.ts` 统一改用 smart patch / smart diff stat，避免每个工具各自处理 whitespace-only 边界。

## 关键代码路径

- `packages/core/src/tools/diffOptions.ts`: smart patch 和 smart structured diff。
- `packages/core/src/tools/edit.ts`: edit confirmation diff 与 diff stats。
- `packages/core/src/tools/notebook-edit.ts`: notebook edit diff。
- `packages/core/src/tools/write-file.ts`: write file diff。
- `packages/core/src/tools/shell.ts`: shell-generated file change diff。
- `packages/core/src/tools/modifiable-tool.ts`: user modification diff stat。

## 验证方式

- `packages/core/src/tools/diffOptions.test.ts`: whitespace-only patch、mixed content/whitespace、identical content、model/user diff stats。
- `packages/core/src/tools/modifiable-tool.test.ts`: diff stat 在 whitespace-only edits 下不再全零。
- 相关工具测试通过 shared helper 间接覆盖 edit/write/notebook/shell 的展示路径。
