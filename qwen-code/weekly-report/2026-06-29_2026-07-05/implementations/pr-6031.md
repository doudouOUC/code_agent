## 解决的问题

open：`qwen channel start` 仍是 standalone ACP-backed 路径，daemon 模式尚不能直接用 `qwen serve --channel` 托管 channel worker，也缺少由 daemon 统一管理 pidfile、status、stop 和 worker lifecycle 的能力。

## 当前实现观察

PR 分支计划让 `qwen serve` 接受 repeatable `--channel <name>` 和 `--channel all`，runtime ready 后启动 serve-owned out-of-process channel worker。worker 经 TypeScript SDK 与 `DaemonChannelBridge` 回连当前 daemon，并复用一个 `SessionRouter` 管理所选 channel；session 创建和 load 强制 `sessionScope: 'thread'`，避免 channel worker 串到默认 shared session。

配套路径包括 `packages/cli/src/commands/channel/daemon-worker.ts`、`runtime.ts`、`pidfile.ts`、`start/status/stop.ts`，以及 `packages/cli/src/serve/channel-worker-supervisor.ts`、`channel-selection.ts`、`run-qwen-serve.ts` 和 `/daemon/status` 的 `runtime.channelWorker` 快照。文档同步涉及 `docs/developers/channel-plugins.md`、`docs/developers/daemon/15-channel-adapters.md`、`docs/users/features/channels/overview.md` 与 `docs/users/qwen-serve.md`。
