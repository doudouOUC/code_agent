## 解决的问题

subagent/background agent token 展示改为 output tokens，避免把 API 吞吐量显示成百万 token。

## 最终实现方案

CLI/web-shell/webui 多个 agent card/status 面统一读 `outputTokens`，旧 payload fallback 到 `totalTokens`。

## 关键代码路径

- `packages/core/src/agents/background-agent-resume.ts`、`background-tasks.ts`、`packages/core/src/tools/agent/agent.ts`: 在 background/subagent 任务结果中保留 output token 计数。
- `packages/cli/src/ui/components/background-view/LiveAgentPanel.tsx`、`BackgroundTasksDialog.tsx`、`messages/InlineParallelAgentsDisplay.tsx`、`messages/ToolMessage.tsx`: CLI/TUI 展示从累计吞吐量改为输出 token。
- `packages/web-shell/client/components/messages/tools/SubAgentPanel.tsx`、`ParallelAgentsGroup.tsx`、`TasksStatusMessage.tsx`: Web Shell subagent/parallel agent 展示同一 output-token 口径。
- `packages/webui/src/components/toolcalls/AgentToolCall.tsx`、`shared/types.ts`: WebUI 工具调用卡片接收并展示 output token 字段。
