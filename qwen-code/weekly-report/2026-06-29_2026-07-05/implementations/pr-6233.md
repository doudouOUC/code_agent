## 解决的问题

daemon 的 `GET /workspace/skills` 状态面已经能区分 SKILL frontmatter 中的 `disable-model-invocation`，但没有把用户 settings 里的 `skills.disabled` 透出给客户端。结果是用户在 `/skills` 对话框中禁用的 skill 仍可能出现在 Web Shell slash-command autocomplete、webui command palette 和 workspace skills 映射结果中，状态协议也和 MCP/tools/hooks 等已有 `disabled` 字段的资源不一致。

## 最终实现

`packages/acp-bridge/src/status.ts` 和 `packages/sdk-typescript/src/daemon/types.ts` 给 workspace skill wire type 增加 optional `disabled?: boolean`。`packages/cli/src/serve/workspace-skills-mapping.ts` 在 `mapSkillConfigToStatus()` 中读取 `disabledSkillNames`，命中时把 `status` 设为 `disabled` 并只在用户禁用场景输出 `disabled: true`，未禁用 skill 不带该字段，保持 additive wire 兼容。

客户端侧不再只看 `status === 'ok'`，而是按 `disabled` 字段过滤用户禁用项：Web Shell 加载 slash skill 列表时过滤 `s.disabled`，`packages/webui/src/daemon/session/mappers.ts` 和 `useDaemonSkills()` 也过滤 disabled skill，使 autocomplete、command list 和 hook 返回值一致。PR 还补了 `acpAgent.test.ts` 的 `getDisabledSkillNames` mock，避免测试 fallback 把缺失 mock 误吞成空列表。

## 关键代码路径

- `packages/acp-bridge/src/status.ts`: `ServeWorkspaceSkillStatus.disabled` wire 字段。
- `packages/cli/src/serve/workspace-skills-mapping.ts`: `skills.disabled` 设置映射到 `status: disabled` 与 `disabled: true`。
- `packages/sdk-typescript/src/daemon/types.ts`: TS daemon SDK 类型补齐 optional `disabled`。
- `packages/web-shell/client/App.tsx`: slash skill 加载与刷新时过滤 disabled skill。
- `packages/webui/src/daemon/session/mappers.ts`: workspace skills 映射过滤 disabled skill。
- `packages/webui/src/daemon/workspace/hooks/useDaemonSkills.ts`: hook 返回给 UI 的 skills 过滤 disabled skill。

## 验证方式

- `packages/cli/src/serve/workspace-skills-mapping.test.ts`: 用户 disabled skill 映射为 `disabled: true`，未禁用项不带该字段。
- `packages/cli/src/serve/workspace-skills-status.test.ts`: status provider 透传 disabled state。
- `packages/webui/src/daemon/session/mappers.test.ts`: disabled skill 不进入 commands/skills。
- `packages/cli/src/acp-integration/acpAgent.test.ts`: ACP workspace skills mock 覆盖 `getDisabledSkillNames`。
- PR body 建议的手工验证是把 skill 名加入 `.qwen/settings.json` 的 `skills.disabled`，确认 `/workspace/skills` 输出 disabled 且 Web Shell `/` 自动补全不再显示。
