## 解决的问题

Web Shell 可以查看、重命名、归档和删除 daemon sessions，但不能直接保存某个会话 transcript。CLI 已有 `/export` 能力，daemon 缺少只读、附件式的 REST 导出面，也缺少 SDK 和 Web Shell 按 capability 暴露下载操作。

## 最终实现

serve capability registry 新增 `session_export`。`GET /session/:id/export` 接受 `format=html|md|json|jsonl`，默认 HTML；非法格式返回 `400 invalid_export_format`，missing session 继续走既有 `SessionNotFoundError` 语义。路由调用 `exportSessionTranscript()`，通过 `SessionService(workspaceCwd).loadSession()` 读取 active persisted session，复用 CLI export 的 `collectSessionData()`、`normalizeSessionData()` 和 `toHtml` / `toMarkdown` / `toJson` / `toJsonl` formatter，直接返回 attachment，不在 daemon host 上写文件。

CLI export util 抽出 chat record tool-call id 归一化 helper，并让 collect/normalize 支持从 persisted conversation records 构造 export data。TypeScript daemon SDK 新增 `DaemonClient.exportSession()` 与类型，WebUI workspace actions 透传该能力。Web Shell sidebar 在 daemon 广告 `session_export` 时显示 download icon，调用 SDK 后用 `Blob` + Object URL 触发浏览器下载，并复用侧栏错误处理。
