## 解决的问题

serve fast path 增加 bundle-level guard。

## 最终实现方案

将真实 esbuild metafile 静态 reachability 检查移到显式 CI script，禁止 ACP/core/vendor 静态闭包进入 pre-listen path。

## 关键代码路径

- `scripts/check-serve-fast-path-bundle.js`: 读取 esbuild metafile，检查 serve fast path bundle closure，禁止 ACP/core/vendor 等重依赖进入 pre-listen path。
- `scripts/tests/serve-fast-path-bundle-check.test.js`: 对 bundle checker 的 positive/negative cases 做脚本级测试。
- `.github/workflows/ci.yml`、`package.json`: 将 fast-path bundle closure check 挂入 CI/脚本入口。
- `packages/cli/src/serve/fast-path.test.ts`、`packages/cli/vitest.config.ts`: 保留 source-level fast path 回归，并让 vitest 配置支持相关检查。

## 验证方式

- `node scripts/check-serve-fast-path-bundle.js`: 在构建产物/metafile 上检查 serve fast path bundle closure。
- `node --test scripts/tests/serve-fast-path-bundle-check.test.js`: 验证 checker 自身的路径匹配与失败报告。
- `cd packages/cli && npx vitest run src/serve/fast-path.test.ts`: 验证 source import closure。
