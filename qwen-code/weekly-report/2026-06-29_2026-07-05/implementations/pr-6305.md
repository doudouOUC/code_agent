## 解决的问题

daemon session 列表只有 recency 和 display-name metadata。长时间使用后，用户无法把会话按项目内主题分组，也无法置顶重要会话；Web Shell 只能展示 legacy recent list。需要在不破坏默认 `/workspace/:id/sessions` 契约、archive/load 边界和 transcript 存储的前提下，提供 opt-in 的分组与置顶能力。

## 最终实现

新增 `session_organization` capability。核心侧增加 `SessionOrganizationService`，把 group catalog 和 per-session organization 存在项目存储目录的 `session-organization.v1.json` sidecar 中，使用 `atomicWriteJSON()` 写入，并用 path-level promise lock 串行化同一 store 的读改写。group name 做 trim、长度和控制字符校验，color 限定在 `red|orange|yellow|green|blue|purple`，重复 name 以大小写折叠判断；删除 group 时会把相关 session 变回 ungrouped。

REST 增加 group CRUD 和 `PATCH /session/:id/organization`。默认 `GET /workspace/:id/sessions` 继续返回 legacy recent list，不带 organization 字段；只有 `view=organized` 时才合并 sidecar snapshot，按 pinned first、group/order 和更新时间排序，并返回 opaque cursor。ACP Streamable HTTP 加入对应 vendor methods，保持 path 参数优先于 body 中冲突字段。TypeScript SDK 增加 group/session organization 方法和 route table 映射。

WebUI/WebShell 通过 capability gate 使用 organized list。Web Shell sidebar 增加 grouped view、pin toggle 和每 session 的 group menu，移动分组使用主题化菜单而不是 browser prompt/native select。
