## 解决的问题

`qwen serve` daemon 与 `qwen --acp` child 之间的 ACP stdio 通路使用 NDJSON。旧 receive path 在处理跨 chunk 的大消息时会反复对不断增长的 pending buffer 做整段 split，最坏情况下呈二次增长；同时 daemon 缺少 event-loop lag、ACP child lag 和 daemon-child pipe payload size 的基础观测面，过载时难以判断是事件循环 stall、pipe 大包还是 child 侧阻塞。

## 最终实现

`packages/acp-bridge/src/ndJsonStream.ts` 新增本地 incremental NDJSON stream。它逐 chunk 扫描换行，pending 只保存跨 chunk 尾段；完整 line 到达后再 decode/JSON.parse，空行忽略，非法 JSON 记录错误后继续处理后续 line，EOF 尾巴丢弃。写出方向手动 `JSON.stringify` + `\n` framing。`NdJsonStreamHooks` 在收发成功时上报 payload bytes，hook 异常被吞掉，避免 metrics 破坏传输。

daemon 专用路径改用这套 stream：`spawnChannel.ts` 支持 `createStream` 注入；`acpAgent.ts` 在 daemon child 模式使用 `ndJsonStream` 并启动 ACP child event-loop monitor，非 daemon 路径继续走 SDK stream。`run-qwen-serve.ts` 在 daemon 进程启动 event-loop lag monitor，注册 OTel gauge，并维护 inbound/outbound pipe byte 统计；`daemon-status.ts` 通过 optional `runtime.perf` 暴露 daemon 进程 event-loop snapshot 与 pipe 统计，child event-loop lag 只通过 telemetry 和 forwarded stderr stall warning 暴露，不塞进 `/daemon/status`。

Telemetry 侧新增 `event-loop-lag.ts`、`event-loop-lag-metrics.ts` 和 daemon pipe histogram：event-loop snapshot 读 `monitorEventLoopDelay` 的 mean/p50/p99/max，stall callback 只在出现新的超过阈值的 max 时触发；OTel gauge 分别注册 `qwen-code.daemon.event_loop.lag` 和 `qwen-code.acp.event_loop.lag`，pipe histogram 记录 `qwen-code.daemon.pipe.message_bytes` 并带 inbound/outbound direction。

## 关键代码路径

- `packages/acp-bridge/src/ndJsonStream.ts`: incremental NDJSON parser/writer 与 byte hooks。
- `packages/acp-bridge/src/spawnChannel.ts`: daemon-only stream factory 注入点。
- `packages/cli/src/acp-integration/acpAgent.ts`: daemon child 使用本地 stream，并启动 ACP event-loop lag monitor。
- `packages/cli/src/serve/run-qwen-serve.ts`: daemon event-loop monitor、pipe stats、perf snapshot wiring。
- `packages/cli/src/serve/daemon-status.ts`: `runtime.perf` optional status shape。
- `packages/core/src/telemetry/event-loop-lag.ts`: event-loop delay snapshot 与 new-max stall callback。
- `packages/core/src/telemetry/event-loop-lag-metrics.ts`: daemon/ACP event-loop OTel gauges。
- `packages/core/src/telemetry/daemon-metrics.ts`: daemon pipe message byte histogram。
- `docs/developers/daemon/19-observability.md`: 新 observability surface 文档。
