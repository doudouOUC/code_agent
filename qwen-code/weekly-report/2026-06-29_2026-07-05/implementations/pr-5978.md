## 解决的问题

引入 adapter-facing `ChannelAgentBridge` 抽象。

## 最终实现方案

adapters/router/channel start 依赖窄 bridge contract；`AcpBridge` 继续作为 standalone 实现，并修 session cleanup、restore/create race 和 bridge swap listener。

## 关键代码路径

- `packages/channels/base/src/ChannelAgentBridge.ts`: 新增 adapter-facing bridge contract，抽出 start/prompt/session lifecycle 所需最小接口。
- `packages/channels/base/src/AcpBridge.ts`、`DaemonChannelBridge.ts`: 让 standalone ACP bridge 与 daemon bridge 都实现同一窄接口。
- `packages/channels/base/src/SessionRouter.ts`、`ChannelBase.ts`: channel adapter 和 session routing 只依赖 `ChannelAgentBridge`，并处理 bridge swap、session cleanup、restore/create race。
- `packages/channels/{dingtalk,feishu,qqbot,telegram,weixin}/src/*`: 各 channel adapter 迁移到 bridge 抽象。
- `packages/cli/src/commands/channel/start.ts`: standalone channel start 使用新抽象装配 bridge。

## 验证方式

- `packages/channels/base/src/ChannelBase.test.ts`: adapter 生命周期和 bridge 接入。
- `packages/channels/base/src/SessionRouter.test.ts`: session routing、cleanup、restore/create race。
- `packages/channels/base/src/sanitize.test.ts`: channel 输出清理不受 bridge 抽象影响。
- `packages/channels/feishu/src/adapter.test.ts`、`qqbot/src/send.test.ts`、`telegram/src/TelegramAdapter.test.ts`: 主要 channel adapter 回归。
- `packages/cli/src/commands/channel/start.test.ts`: CLI channel start 的 bridge 装配。
