## 解决的问题

open：`/skills` 在 interactive 模式打开 dialog，但 ACP / non-interactive 环境没有 dialog UI，只能返回可读文本。现有 fallback 信息量不足，未展示 skill description 和 level，也容易与 dialog 列表的排序、禁用过滤和本地化标签不一致。

## 当前实现观察

PR 分支让 `skillsCommand` 保持 interactive 下直接返回 `{ type: 'dialog', dialog: 'skills_manage' }`；ACP / non-interactive 下读取 `skillManager.listSkills()`，过滤 `userInvocable === false` 和 `skills.disabled`，按 `normalizeSkillPriority` 降序、name 升序排序，然后输出：

```text
Available skills:

  - skill-name  description  (Project/User/Extension/Bundled)
```

`packages/cli/src/ui/utils/skill-level-label.ts` 抽出 `levelLabel()`，dialog list 与 ACP text 共用本地化 level 文案；description 会压成单行并截断，裸 skill（无 description/level）不会留下空括号或多余尾随空格。

## 关键代码路径

- `packages/cli/src/ui/commands/skillsCommand.ts`: ACP/non-interactive listing fallback。
- `packages/cli/src/ui/components/views/SkillsList.tsx`: dialog list 复用 `levelLabel`。
- `packages/cli/src/ui/utils/skill-level-label.ts`: project/user/extension/bundled 标签。
- `packages/cli/src/ui/types.ts`: skill level 字段收敛。

## 验证方式

- `packages/cli/src/ui/commands/skillsCommand.test.ts`: interactive dialog、ACP fallback、无 description/level、non-user-invocable、disabled skills、submit-on-accept。
- `packages/cli/src/ui/components/skills/SkillsManagerDialog.tsx` 与 `SkillsList.tsx` 通过共享 helper 避免大小写/本地化漂移。
