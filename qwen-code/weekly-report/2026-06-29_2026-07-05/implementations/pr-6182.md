## 解决的问题

`ChannelAgentBridge` 已经为 adapters 提供 create/load/prompt/cancel 与 command 查询能力，但 channel adapter 或诊断工具无法枚举当前 bridge 内附着的 daemon sessions，也看不到某个 session 是否有 active prompt。这限制了运行态观测和 session management。

## 最终实现

`packages/channels/base/src/ChannelAgentBridge.ts` 新增 `BridgeSessionInfo`，包含 `sessionId`、`workspaceCwd` 与 `hasActivePrompt`；`ChannelAgentBridge` 增加 optional `listSessions()`。方法保持 optional，避免要求所有旧 bridge 或测试 double 立刻实现。

`DaemonChannelBridge.listSessions()` 从内部 `sessions` map 构造 point-in-time snapshot，并用 `activePrompts` set 填充 `hasActivePrompt`。drop/cancel/stop、same-id replacement 和 prompt finally 已维护这些内部集合，因此 listing 能反映当前可路由 session。`createDaemonChannelBridgeFacade()` 在 bridge 支持时透传 `listSessions()`，与已有 optional `getAvailableCommands()` 模式一致。

## 关键代码路径

- `packages/channels/base/src/ChannelAgentBridge.ts`: `BridgeSessionInfo` 与 optional `listSessions()` contract。
- `packages/channels/base/src/DaemonChannelBridge.ts`: sessions snapshot 与 `hasActivePrompt` 标记。
- `packages/channels/base/src/index.ts`: 类型导出。
- `packages/cli/src/commands/channel/daemon-worker.ts`: daemon-worker facade optional forwarding。
