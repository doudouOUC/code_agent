## 解决的问题

standalone 包中的 `qwen serve` 走 `cli-entry.js` fast path。

## 最终实现方案

打包脚本让 serve shim 调 `lib/cli-entry.js`，非 serve 保持 `node --expose-gc lib/cli.js`，并跳过 npm-only artifacts。

## 关键代码路径

- `scripts/create-standalone-package.js`: 生成 standalone 包时为 serve shim 指向 `lib/cli-entry.js`，保留非 serve 命令的 `lib/cli.js` 路径，并过滤 npm-only artifacts。
- `scripts/tests/install-script.test.js`: 覆盖 standalone package 生成结果，验证 serve shim 和普通 CLI entry 的分流。

## 验证方式

- `node --test scripts/tests/install-script.test.js` 或仓库脚本中的 install-script 测试：检查 standalone package 中 serve shim 的入口路径。
- 关键人工检查：生成包内 `qwen serve` 不经 `lib/cli.js`，避免启动时拉入完整 CLI/runtime graph。
