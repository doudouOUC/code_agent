## 解决的问题

Web-shell/daemon 会话缺少 `/settings` slash command，用户在 WebShell 侧无法像 TUI 一样直接查看或进入设置相关流程。

## 最终实现方案

新增 `/settings` 在 serve / ACP / web-shell 路径中的命令接线，补齐命令输出、类型和前端消费链路；对应文件覆盖 serve command handling、settings command 适配、web-shell UI/状态处理以及相关测试，使 daemon-backed 客户端能通过同一 slash command surface 访问设置能力。
