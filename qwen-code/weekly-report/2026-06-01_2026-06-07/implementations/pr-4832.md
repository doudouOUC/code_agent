## 解决的问题

daemon/web-shell 缺少 extensions 诊断 surface，客户端难以查看当前 workspace/session 的扩展加载、可用性和诊断状态。

## 最终实现方案

新增 extensions diagnostic HTTP/ACP surface：在 serve 侧暴露 workspace/session 扩展诊断入口，ACP slash command 接入 `/extensions`，并补齐 capability、SDK/types、事件或响应结构与服务端测试。最终让 daemon-backed WebShell 可以读取扩展状态而不需要依赖 TUI 内部实现。
