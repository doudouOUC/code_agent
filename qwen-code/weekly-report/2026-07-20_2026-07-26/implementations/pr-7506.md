## 解决的问题

enterprise memory runtime/management service 已经提供安全边界后，还需要一个 Qwen 侧集成层：既能在不阻塞 coding request 的前提下捕获 bounded lifecycle/tool evidence，又不能把企业 broker credential、管理权限、删除/批准/策略操作暴露给模型或工具环境。

## 当前实现方案

当前 open diff 通过 opt-in signed extension、fail-open hooks 与最小 MCP surface 连接外部 Memory Gateway。Hook handler 捕获 session/tool lifecycle evidence，遇到 Hook timeout、gateway outage、本地 state-store 失败或 circuit breaker 时不阻塞 Qwen request，而是记录 bounded retry state。Agent 通过 externally signed launcher 获得 per-operation signed material，企业 broker/signing config 不写入 Qwen settings 或 tool-visible env。

MCP server 只暴露 search/get/propose/advisory feedback 等非管理面能力；写入要求 caller-supplied UUID operation id 跨 reconnect 稳定，read request id 保持 connection-unique，exact retry 必须匹配同一 operation fingerprint。extension manifest 不提供 delete/approve/activate/promote/policy/tenant-selector/principal-selector/repository-selector 等管理工具。

## 关键代码路径

- `integrations/enterprise-memory/src/agent/gateway-client.ts`: runtime gateway client 与 signed request material。
- `integrations/enterprise-memory/src/agent/hook-handler.ts`: fail-open Hook evidence capture、bounded retry/circuit behavior。
- `integrations/enterprise-memory/src/agent/mcp-server.ts`: narrowed MCP search/get/propose/feedback surface。
- `integrations/enterprise-memory/src/agent/state-store.ts`: deterministic local retry state。
- `integrations/enterprise-memory/hooks/hooks.json`, `qwen-extension.json`: Hook 与 extension manifests。
- `integrations/enterprise-memory/README.md`: deployment guidance。
