## 解决的问题

企业团队需要跨当前授权用户共享长期记忆，但直接把公共 memory MCP server 接进 Qwen Code 无法可靠提供 tenant/principal/workspace/repository 身份边界、生命周期证据采集、review-before-share、删除一致性、防投毒、防复活、保留策略和审计。该 PR 尝试一次性提交完整 opt-in enterprise memory integration，并保持 Qwen Core 不改。

## 关闭前实现方案与拆分去向

关闭前单体 diff 新增 `integrations/enterprise-memory` 工作区包：PostgreSQL/RLS 作为 canonical store，Mem0 仅作为可替换 semantic index；gateway 负责 trusted identity、runtime capability、idempotency、consent/review、retention/audit/deletion/anti-resurrection；Qwen 侧通过 signed extension、fail-open hooks 与小 MCP surface 捕获证据和召回候选。它还包括 OpenAPI、SQL migration、management/runtime HTTP server、Agent/hook/MCP adapter、state store、deployment docs 和 141 个测试。

该 PR 未合入，随后被拆成 stack：#7509 foundations、#7505 canonical persistence、#7507 governed lifecycle、#7508 runtime/management APIs、#7506 Qwen Agent integration。最终文档以拆分后的 stack 为准，#7502 只作为单体方案关闭记录。
