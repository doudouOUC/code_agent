## 解决的问题

在评审 enterprise memory 的存储和运行行为前，需要先有可审查的 trust/type foundation：tenant、principal、workspace、repository、runtime binding、request capability、内容保护、bounded HTTP parsing 和 semantic-index contract 必须独立成层，且不能把企业 identity provider、policy system 或 semantic provider 放进 Qwen Core。

## 当前实现方案

当前 open diff 新增 `integrations/enterprise-memory` workspace package 并接入根 build/typecheck。基础层定义 domain model、strict config、bounded JSON/HTTP parser、content protector、runtime binding authorizer、request-bound capability verifier/replay store 和 provider-neutral semantic index interface。capability token 在使用前绑定 exact request、tenant、principal、workspace、repository、lease epoch 与 replay identity；config 拒绝 malformed URL、不安全 production endpoint 和非法 numeric limits。

该 PR 还包含总体设计记录，但不引入 canonical database、memory lifecycle、runtime/management server 或 Agent/MCP integration；这些在后续 #7505/#7507/#7508/#7506 中分层实现。

## 关键代码路径

- `integrations/enterprise-memory/src/domain.ts`: tenant/principal/workspace/repository/scope 类型。
- `integrations/enterprise-memory/src/config.ts`, `http-json.ts`: strict config 与 bounded HTTP parsing。
- `integrations/enterprise-memory/src/content-protector.ts`: memory content protection boundary。
- `integrations/enterprise-memory/src/runtime-binding-authorizer.ts`: immutable runtime binding auth。
- `integrations/enterprise-memory/src/security/capability-verifier.ts`, `request-binding.ts`: request-bound capability validation。
- `integrations/enterprise-memory/src/capability-replay-store.ts`: idempotency/replay control。
- `integrations/enterprise-memory/src/semantic-index.ts`: provider-neutral semantic index contract。
