## 解决的问题

工具输出过去在多个互不共享状态的层面被截断：Shell producer 约 30K、generic tool output 约 2K、Core scheduler batch budget 再尝试 aggregate offload。scheduler 通过人类可读 truncation marker 判断“已经截断”，结果多个已截断 Shell/MCP 结果合起来仍可能超过 aggregate budget。

更大的缺口是运行时边界不统一。headless 每个 tool call 都有自己的 scheduler，之后在外层拼接 response；interactive 会在 scheduler finalization 后追加 duplicate/synthetic responses；ACP、Agent runtime 和 speculative follow-up 也在 scheduler 外组装 tool batch。最终可能出现：发给模型的 tool result 超预算、同一输出被重复持久化、录进 transcript 的 parts 和下一次请求里的 parts 不一致。

## 最终实现方案

最终实现为 tool result 增加内部 `persistedOutputFiles` 元数据，挂在 `ToolResult` 和 `ToolCallResponseInfo` 上。`undefined` 表示 producer 没做持久化判断，`[]` 表示判断过但没有可复用文件，非空数组表示已有 producer artifact。该字段不进入 hook serialization、ACP payload、JSON output、telemetry attributes 或 UI metadata。

Shell/MCP/generic producer 被拆成“完整输出持久化”和“模型预览”两件事。Shell 仍保留约 30K trigger，但模型预览压到约 4K head/tail；MCP large output 保留完整 transformed rich display，同时给模型约 2K preview；generic `persistAndTruncateToolResult()` 返回实际写入 path，primary/fallback writer 都能供后续 finalizer 复用。

新增 `packages/core/src/utils/tool-response-finalizer.ts`。`finalizeToolResponses(config, entries)` 收集 original-order response parts 中的 top-level text、`functionResponse.response.output` 和 `error` 字段，排除成功 `enter_plan_mode` 的 lifecycle policy output；当总长度超过 `toolOutputBatchBudget` 时，用 max-min water-fill 给每个 text slot 分配字符预算。需要裁剪且尚无 artifact metadata 的 entry 会先尝试持久化一次；失败不阻止 final no-I/O hard cap。裁剪文本保留 artifact path、head/tail preview，且不拆 Unicode surrogate pair。最终 `toolResponseTextLength()` 重新计算 `contentLength`。

runtime boundary 统一收敛：Core scheduler 在 `PostToolBatch` hook 前 finalize，避免 hook 输入失控；hook 可能改写 response 后再 finalize 一次。interactive hook 把 executable、duplicate、synthetic response 按原始 call order 合并后再提交和录制；headless 在外层收集 duplicate/skipped/cancelled/executed 后一次 finalize；ACP 在完整 tool-call turn 录制前 finalize；Agent runtime 和 speculative follow-up 在 append history / emit model-facing result 前 finalize；`GeminiChat` 发送边界还有 `enforceFunctionResponseBudget()` 作为 no-I/O safety cap。

`enter_plan_mode` 是唯一语义豁免：成功 output 安装 Plan mode lifecycle policy，不能因预算被截断；失败 text 和同批其它 ordinary output 仍受预算控制。
