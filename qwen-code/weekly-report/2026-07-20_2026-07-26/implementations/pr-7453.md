## 解决的问题

#7400 建立了 prompt terminal exactly-once 主路径，但 self-review 发现几个边缘缺口：移除 RUNNING prompt 后会把条目从 pending list 中删掉，若 agent 不配合取消且随后 session teardown，flush 已看不到该 prompt，等待 terminal 的 SSE consumer 会挂起；queued prompt 的 deadline/teardown failure 还会污染 session-level `turnError` 与 retry state；queued deadline 在 pre-dispatch abort 检查里会以普通 `AbortError` 拒绝；`prompt-deadline` helper 还通过 re-export 把 bridge 拉进叶子模块。

## 最终实现方案

RUNNING prompt 的 remove 不再立即从 `pendingPromptList` 物理删除，而是打上 removed/hidden 标记：pending-prompts API 看不到它，重复 remove 幂等返回未移除，但 teardown flush 仍能在 agent settle 前看到并发布 terminal。settle path 最终负责清理该条目，并通过 terminal latch 抑制重复 completed 事件。

`publishPromptTerminal()` 更新 session-level turn state 时改看 prompt 自身是否 running：queued prompt 的 deadline 或 teardown terminal 只发事件，不设置 session `turnError`/`retryAllowed`。pre-dispatch abort 检查传播 `PromptDeadlineExceededError`，使 queued 与 running deadline 对调用方呈现同一 typed error。`PromptDeadlineExceededError` 的 server 侧导出改经 bridge 边界，`server/prompt-deadline.ts` 恢复为纯 helper 叶子模块。

## 关键代码路径

- `packages/acp-bridge/src/bridge.ts`: removed-running prompt 留在 pending list、queued failure 不污染 turn state、deadline typed rejection、debug-level duplicate terminal log。
- `packages/acp-bridge/src/bridgeTypes.ts`: pending prompt removed/hidden 状态字段。
- `packages/acp-bridge/src/bridge.test.ts`: running prompt remove + close terminal、queued deadline typed error、session summary 不被 queued failure 污染。
- `packages/cli/src/serve/server.ts`, `packages/cli/src/serve/server/prompt-deadline.ts`: deadline error 导出边界调整。
