## 解决的问题

#7323 引入 `persistedOutputFiles` 三态元数据后，Shell producer 在“完整输出保存失败、但已经生成有界模型预览”的分支必须返回空 artifact list `[]`，而不是省略 metadata。否则 finalizer 会把该 producer 视为尚未做持久化判断，可能重复尝试持久化。运行时行为已经正确，但这个无 artifact 分支缺少 producer 级回归测试，类型文档也没有解释空数组 sentinel。

## 最终实现方案

`packages/core/src/tools/shell.test.ts` 新增回归用例：mock truncation helper 返回短预览但无 output file，断言 Shell tool 保留短预览、从 model-facing content 中移除原始超长正文，并把 `persistedOutputFiles` 设置为 `[]`。`packages/core/src/tools/tools.ts` 补充 `persistedOutputFiles` 三态注释，明确 `undefined`、`[]`、非空数组分别代表未判断、已判断无 artifact、已有可复用 artifact。

## 关键代码路径

- `packages/core/src/tools/shell.test.ts`: Shell truncation without artifact 回归测试。
- `packages/core/src/tools/tools.ts`: `persistedOutputFiles` internal metadata 三态契约注释。
