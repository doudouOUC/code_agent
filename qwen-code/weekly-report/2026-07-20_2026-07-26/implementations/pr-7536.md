## 解决了什么问题

qwen-code 既有 LLM、tool、agent span 使用一组私有字段，和 OpenTelemetry GenAI semantic conventions 以及阿里云 ARMS LLM Trace 的字段名、类型、含义不完全一致。结果是 provider、operation、conversation、response model、finish reason、cache usage、tool-call id、agent description 等字段无法被后端按统一语义检索或展示；同时不同 provider 的响应模型、finish reason 和 cache token provenance 也容易在转换链路中丢失。

## 最终实现

最终实现新增 `gen-ai-provider.ts`，集中解析 provider 名称、auth type、operation name 与 output type，并用 hostname 精确匹配防止伪造后缀。`gen-ai-usage.ts` 用 WeakMap 给 usage metadata 附加 cache provenance，不改变公开 usage shape。OpenAI、Anthropic、Gemini/Qwen 的 converter、pipeline、usage builder 和 logging content generator 都保留 provider response model、finish reason、cache read/write、response id 与 provider tool-call id。

`session-tracing.ts` 扩展 LLM/tool/subagent span 属性：保留原私有字段，同时写入与 GenAI/ARMS 对齐的 `gen_ai.*`、conversation、response、finish reason、tool-call 和 agent description 字段。`loggingContentGenerator` 在流式/非流式路径按 candidate index 稳定排序 finish reasons，aborted partial stream 也保留已知 response metadata。测试覆盖 OpenAI、Anthropic、fake OpenAI integration、ACP session、core tool scheduler、agent tool 和 provider resolution。
