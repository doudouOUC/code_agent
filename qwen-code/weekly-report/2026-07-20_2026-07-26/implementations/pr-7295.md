## 解决的问题

ACP session 在等待权限时可能同时遇到父级 prompt cancellation。旧路径把 permission cancel、loop/Stop-hook terminal path 或 background notification 权限等待统一收束成 `end_turn`，即使 abort signal 已经被触发。这样 host 侧看到的是正常 turn 结束，而不是用户取消。

另一个问题是 mid-turn message drain：取消发生时，如果先前 timeout 已经把部分 mid-turn message 恢复到内存，旧逻辑可能在 abort catch 中直接返回，导致这些 recovered message 没有被写回 stopped tool run 的上下文。

## 最终实现方案

PR 在 `Session.ts` 中新增 `getAbortAwareEndTurnStopReason(signal)`。所有在 tool run 之后可能以 terminal path 结束的分支，返回前都用该 helper 读取当前 abort state：如果 parent signal 已 aborted，stopReason 返回 `cancelled`；否则保持 `end_turn`。主 prompt loop 在顶部发现 `pendingSend.signal.aborted` 时也直接返回 `cancelled`，并 suspend Todo stop guard。

`#preserveStoppedToolRun()` 不再无条件调用 `#drainMidTurnUserMessages()`。它在 abort 场景中先取 `#takeRecoveredMidTurnMessages()`，再以 `{ preserveFallbackOnAbort: true }` 调用 `#buildMidTurnParts()`。`#buildMidTurnParts()` 在 abort 且允许 fallback 时不会直接丢弃 message，而是回退成原始 display text；非 abort 错误才记录 warning。这保证取消响应里仍能保留已恢复的 mid-turn 用户输入。

background notification 路径同样在 permission cancel 和 loop detected 后使用 abort-aware stop reason。Plan-mode unknown shell approval、Stop hook permission、Stop-hook iteration 间 abort、background notification awaiting permission 都有针对性单测。

## 关键代码路径

- `packages/cli/src/acp-integration/session/Session.ts:getAbortAwareEndTurnStopReason`: parent cancellation 优先于 simultaneous terminal path。
- `Session.prompt()` 主循环与 Stop-hook/permission terminal branches: abort 时返回 `cancelled`。
- `Session.#preserveStoppedToolRun()`: abort 场景保留 recovered mid-turn message。
- `Session.#buildMidTurnParts()`: 新增 `preserveFallbackOnAbort` option，abort 时可用原始 display text 兜底。
- background notification loop: permission cancel / loop path 用 abort-aware end-turn emission。

## 验证方式

- `packages/cli/src/acp-integration/session/Session.test.ts`: unknown Plan shell approval pending 时取消，断言 prompt 结果为 `cancelled`、approval onConfirm 收到 Cancel、tool 未执行、recovered mid-turn message 被记录。
- Stop hook permission pending 时取消，断言 `stopReason: cancelled` 且 tool 未执行。
- Stop-hook 两次迭代之间 abort，断言 second stream 后返回 cancelled。
- background notification 等待权限时取消，断言通知结束 reason 保持 cancellation。
- 既有 permission cancel、Todo stop guard、mid-turn drain 和 background notification tests 继续覆盖非 abort 行为。
