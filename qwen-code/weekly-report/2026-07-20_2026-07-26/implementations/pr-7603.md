## 解决了什么问题

#7463 新增 Java daemon transport 后，还需要跟上 #7458 的 restart-safe event cursor 契约，并收紧若干 Java-side reliability edge：prompt admission 需要把 `lastEventId` 与 `eventEpoch` 作为一组保存和回传，SSE response epoch mismatch 不能继续消费事件，JSON/SSE malformed path 不能悄悄推进 cursor，terminal-before-202 与 teardown ordering 也需要端到端验证。

## 最终实现

当前 open diff 扩展 `PromptAcceptance`，记录 optional `eventEpoch`；`DaemonClient` 和 `DaemonSessionClient` 在 prompt admission、初始 SSE request 和 reconnect 时发送 `X-Qwen-Event-Epoch`，并能从 validated SSE response 学习 epoch。若 daemon 返回不同 epoch，Java client 在消费事件前 fail closed；若 response 缺 header，则保留已知 epoch，不把未知值覆盖掉。

SSE/JSON 层继续 harden：`SseReader` 处理 event id gap、compressed stream 和 invalid frames；`JsonSupport` 增加 truncated JSON / null data 等严格测试；bridge 侧补 terminal-before-202 buffering 与 teardown 顺序测试，确保 Java pending prompt 不会错过正式 terminal。E2E harness 增强 restart/event epoch 场景，Java CI/release workflow 也补充 daemon transport 测试路径。

## 关键代码路径

- `packages/sdk-java/qwencode/src/main/java/com/alibaba/qwen/code/daemon/PromptAcceptance.java`: admission watermark + event epoch。
- `DaemonClient.java`, `DaemonSessionClient.java`, `HttpSupport.java`: epoch header send/learn、prompt/session reconnect。
- `SseReader.java`, `JsonSupport.java`: strict SSE/JSON reliability boundaries。
- `packages/sdk-java/qwencode/src/test/java/com/alibaba/qwen/code/daemon/*`: session client、SSE reader、JSON support、E2E tests。
- `packages/acp-bridge/src/bridge.ts`, `bridge.test.ts`: terminal-before-response / teardown ordering support。
- `scripts/run-java-daemon-sdk-e2e.ts`: Java daemon E2E harness。
- `.github/workflows/sdk-java.yml`, `release-sdk-java.yml`: Java CI/release coverage。
