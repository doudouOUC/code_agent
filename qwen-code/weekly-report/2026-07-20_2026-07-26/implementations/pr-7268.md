## 解决的问题

daemon 过去只在构造 `WorkspaceRuntime` 时评估 workspace trust。用户 settings、system/default folder trust、IDE trust 或 `trustedFolders.json` 变化后，运行中的 trusted runtime 仍会继续持有旧的 settings、env、filesystem、ACP mount、MCP、extension、channel worker、scheduled task 和 session admission，直到 daemon 重启。

trust 降级不能就地修改运行中 runtime，因为 filesystem factory、bridge、settings、environment、ACP mount 和 workspace-scoped managers 都在构造时捕获输入。安全要求是：trust revoke 后，旧 trusted generation 必须先关闭 admission，再做 drain；失败也不能恢复旧 trusted runtime 或 fallback primary。

## 当前实现方案

当前 open diff 新增 side-effect-free `daemon-trust-policy` loader。它只读取 system override、user settings、system defaults、IDE trust 和 `trustedFolders.json`，不读取 workspace settings 或 project env。loader 产出 immutable semantic snapshot，workspace 再把 snapshot materialize 成 effective trust 和 allowed roots；只有 materialized state 变化才触发 runtime replacement，source-only revision 变化只推进 policy revision。

`daemon-trust-policy-monitor` 每秒重新读取 policy 输入，按 semantic hash 发布变化；IDE trust 和 same-process trusted-folder 写入会触发即时 reload。`trustedFolders.ts` 写入路径加 proper-lockfile、锁内重读、保留 comments、0600 regular file 原子替换和 symlink 防护， malformed file 不会被静默重写。

registry 侧把 workspace 身份拆成 stable `WorkspaceEntry` 与 immutable `RuntimeGeneration`。每个 generation 有 one-way `generationGuard`，trust decrease 时 reconciler 在任何 async drain 之前同步 close affected generation，之后 drain bridge/channel/Voice/sub-session/ACP mount/channel worker/scheduled/git state，再用 fresh settings/env/filesystem/trust/cache 重建 runtime。grant 也走 destructive replacement；构造失败会尽量落到 untrusted runtime，containment 无法确认时 entry 进入 blocked/degraded，不让其它 workspace 受影响。

所有 workspace-qualified data-plane route 都在请求时解析当前 entry/runtime。privileged mutation 捕获 generation guard，并在 commit boundary 前 recheck；ACP、Voice、scheduled task、agent、memory、settings、MCP、extension、Git、goal、contact、filesystem write/edit 都加了 generation/ trust guard。transitioning、failed、blocked、draining 或 stale owner 一律 fail-closed，不 fallback primary。

协议面新增 `workspace_trust_hot_reload` capability。trust status v1 保持兼容；v2 status 分离 configured/effective trust、policy revision、`stable|applying|failed` reconciliation state 和 stable error code。SDK 增加 v2 类型和 helper，Web Shell 在 trust request 后刷新/poll v2 status，并显示 applying/failed state。

## 关键代码路径

- `packages/cli/src/config/daemon-trust-policy.ts`: policy snapshot、settings/trusted-folder parsing、semantic revision、workspace trust evaluation。

- `packages/cli/src/config/trustedFolders.ts`: trusted-folder lock/rewrite/change listener。
- `packages/cli/src/serve/workspace-registry.ts`: `WorkspaceEntry`、`RuntimeGeneration`、`WorkspaceGenerationGuard`、generation-aware owner index。
- `packages/cli/src/serve/workspace-trust-reconciler.ts`: trust replacement planning、generation close、runtime rebuild、failure/blocked state。
- `packages/cli/src/serve/run-qwen-serve.ts`, `server.ts`: topology gate、policy monitor lifecycle、reconciler wiring、shutdown ordering。
- `packages/cli/src/serve/routes/*`, `workspace-*`: workspace route generation capture/recheck 和 trust-unavailable responses。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`, `types.ts`: trust status v2 type/helper。
- `packages/web-shell/client/components/sidebar/WorkspaceSection.tsx`: trust v2 polling、applying/failed UI state。
