## 解决了什么问题

#7458 合入 event epoch 与 degraded replay 后，review follow-up 发现三个小但会影响协议可读性和回归覆盖的问题：HTTP session load route 可能在 response DTO/whitelist 处丢掉 `eventEpoch` 与 `replayDegraded`；compaction degraded breadcrumb 直接插入 session id，不利于日志/文本的安全转义；TS SDK 注释把 `state_resync_required.detail` 写得像任意 index signature，容易误导消费者。

## 最终实现

最终实现补 route-level regression test，直接断言 session load response 中包含 `eventEpoch` 与 `replayDegraded`，把 #7458 的 bridge/event-bus 层字段传播约束钉到 HTTP API 边界。compaction degraded breadcrumb 改用 `JSON.stringify(sessionId)` 输出 session id，使文本可解析、可转义，并避免特殊字符破坏诊断语句。

TS SDK 的 daemon event 注释同步修正为显式 `detail` 字段说明，表达 `state_resync_required` 可以通过 `detail:'epoch_mismatch'` 等 additive detail 补充原因，而不是任意扩展属性。

## 关键代码路径

- `packages/cli/src/serve/server.test.ts`: session load route 回归，覆盖 `eventEpoch` / `replayDegraded` response 字段。
- `packages/acp-bridge/src/bridge.ts`: degraded breadcrumb 的 session id 安全转义。
- `packages/sdk-typescript/src/daemon/events.ts`: `state_resync_required.detail` 注释修正。
