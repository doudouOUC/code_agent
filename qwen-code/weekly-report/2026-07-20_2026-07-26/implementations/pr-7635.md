## 解决了什么问题

#7536 已把 provider、response、usage、finish reason、tool-call 等 GenAI/ARMS metadata 对齐到 span 上，但 request 参数侧仍缺一组 ARMS 可消费字段：choice count、max tokens、temperature、top_p、frequency penalty、presence penalty 和 stop sequences。直接从用户输入或配置读取会不准，因为 provider adapter 会补默认、删除不支持字段、做 output-window clamp，最终发送给 provider 的 request 才是可观测真相。

## 最终实现

最终实现新增 `telemetry/gen-ai-request.ts`，用 OpenTelemetry Context 携带 request observer。OpenAI、Anthropic、Gemini/Qwen 的 content generator 在 provider-final SDK request 发送边界调用对应 report helper，抓取第一份最终 request snapshot，并把可确认的字段写入当前 LLM span。

字段写入采用严格语义：没有出现在最终 request 中的 SDK/server default 不推断；非法或不可表示的值省略；choice count 为 1 时省略以降低噪声；0、空数组等有意义的安全值保留；retry/fallback 后续 attempt 不覆盖第一次 snapshot；telemetry observer failure best-effort，不影响模型请求。

## 关键代码路径

- `packages/core/src/core/anthropicContentGenerator.ts`: Anthropic request 参数提取。
- `packages/core/src/core/geminiContentGenerator.ts`: Gemini/Qwen request 参数提取。

- `integration-tests/cli/gen-ai-telemetry.test.ts`: fake OpenAI/CLI telemetry integration 覆盖。

- `integration-tests/cli/gen-ai-telemetry.test.ts` 验证 CLI 端到端 GenAI request attributes。
