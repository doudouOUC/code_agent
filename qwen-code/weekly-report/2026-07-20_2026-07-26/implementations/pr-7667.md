## 解决了什么问题

#7536/#7635 已对齐 GenAI/ARMS 的 metadata 与 request parameter 字段，但敏感内容字段仍分散在私有 span attributes、tool schema event 和 provider-specific payload 里。后端如果要按 OTel/ARMS 的统一 schema 展示 LLM input/output messages、system instructions、tool definitions、tool call arguments 和 tool call result，需要一套稳定的标准字段转换与敏感属性门控。

## 最终实现

当前 open diff 新增 GenAI content/exchange observer。LLM input、system instructions 和 tool definitions 从第一份 provider-final SDK request 捕获；LLM output messages 从最终物理 request attempt 捕获。OpenAI、Anthropic、Gemini 和 Vertex payload 都转换为 pinned GenAI JSON schema，支持 multi-candidate 与 streaming，但不保留 raw chunk。

Tool span 侧记录三类字段：静态 registry description（非敏感、4096 UTF-16 上限并做 surrogate-safe 截断）、执行前最终 invocation params、成功后的 model-facing result。soft error 不写 result，避免把失败中间态当成可消费输出。旧的 exact-equivalent 私有内容字段与 `tool_schema` event 被删除或降级为标准字段 adapter；deprecated public helpers 仍作为 thin standard-field adapter 保留。

敏感内容继续由 `telemetry.includeSensitiveSpanAttributes` 控制，并受 `sensitiveSpanAttributeMaxLength` 约束。JSON invalid、cyclic、不完整或超过限制时整项省略；tool definitions 需要 type/name，optional parameters 只有能归一到 Draft-07 时才写入。`gen-ai-exchange` 用 span-local observer 隔离并发 span，first input / final output attempt 不串值；即便 content capture 关闭，非敏感 request fields 仍可照常记录。
