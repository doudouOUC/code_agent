## 解决的问题

在 telemetry 与 undici lazy-loading 之后，`@google/genai` 仍留在 ACP bootstrap static closure 中。简单 dynamic import 不够，因为 ACP session 创建会急切构造 provider，SDK 只会从 initialize 移到 `POST /session`；目标是让 session create 不执行官方 SDK，直到首次异步模型操作才加载 provider。

## 当前实现方案

当前 open diff 为 core orchestration 提供 package-local、SDK-parity 的同步 surface，实现不依赖官方 SDK 的 request/content normalization、compat response 类和 small synchronous helpers，同时继续保留官方 SDK 类型与 provider response class 兼容。Provider 构造和 logging decorator 由 single-flight lazy content generator 在第一次 async model operation 时加载；两个首次操作并发时共享同一次 provider import/construct。

配置校验、runtime fetch preparation 和 Qwen OAuth cached credential acquisition 仍保持 eager，避免 session accepted 后才发现 auth 不可用。MCP tool adaptation 仍在 discovery/direct invocation 时加载官方 SDK。bundle guard 增加 `@google/genai` static closure blacklist，要求 ACP eager closure 中 SDK 字节为 0，但 dynamic provider/MCP chunks 仍可包含 SDK。
