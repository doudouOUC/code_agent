## 解决了什么问题

部分私有部署只需要“在一个受信运行环境里搜索外部上下文”的轻量入口，而不是 enterprise memory gateway 的多租户、管理面、review queue 和长期记忆生命周期。风险在于模型不应该选择 provider、corpus、credential，也不应该把 provider 输出当成可信指令；同时当前阶段还没有引入自动 recall、写入记忆或删除/审批等管理动作的安全边界。

## 最终实现

当前 open diff 已收敛为 retrieval-only external context search。`integrations/external-context` 作为独立 extension 暴露一个 MCP tool：`context_search({ query })`。provider、corpus、credential env、timeout、limit 等都由管理员在 MCP server 启动前通过配置固定，模型只能提供 query，不能选择 provider、切换 project/index/corpus，也不能读取 credential 名称。

运行时支持两类 adapter：Mem0 Platform V3 Search 与 Generic HTTP Search V1。Mem0 adapter 固定 `app_id`、`top_k=5`、`threshold=0.1`、`rerank=false` 等搜索选项；Generic HTTP adapter 只向 `/v1/context/search` POST `{query, limit}`。HTTP client 拒绝 redirects，除 loopback 外要求 HTTPS，并对 timeout、abort、invalid JSON/UTF-8、oversized response 和 malformed item 做 strict validation。provider 失败统一返回本地稳定错误 `External context search failed.`，不把 provider error detail、credential 或配置路径泄漏给模型。

部署上保留两个 profile。managed profile 通过 `examples/managed-mcp.json`、`managed-settings.json` 和受控 launcher 固定 MCP config 与 allowed tool，禁止 `/cd` 等会改变仓库边界的行为；manifest 只用于本地可信 trial，不代表托管环境的安全来源绑定。direct profile 信任当前 repo env files 和同 UID 进程，适合单可信仓库，不提供 DLP、身份/文档 ACL、不可绕过确认、合规审计或 prompt-injection 防护；需要这些能力时仍应使用 enterprise memory gateway / governed profile。

## 关键代码路径

- `integrations/external-context/src/config.ts`, `config.test.ts`: strict config、provider selection、credential env、endpoint、timeout 和 managed profile 校验。
- `integrations/external-context/src/mcp.ts`, `mcp.test.ts`: 唯一 MCP surface `context_search({query})`，不注册 recall hook 或 write tool。
- `integrations/external-context/src/providers.ts`, `providers.test.ts`: Mem0 Platform V3 Search 与 Generic HTTP Search V1 adapter。
- `integrations/external-context/src/http-client.ts`: bounded POST JSON、HTTPS/loopback、redirect rejection、timeout/cancel 和 error taxonomy。
- `integrations/external-context/src/runtime.ts`: config → provider binding → MCP tool execution 的运行时装配。
- `integrations/external-context/examples/mem0.json`, `generic-http.json`, `managed-mcp.json`, `managed-settings.json`: direct / managed 配置样例。
- `integrations/external-context/README.md`, `qwen-extension.json`: extension 使用范围、信任边界和安装入口。

## 验证方式

- `npm test --workspace=@qwen-code/external-context` 覆盖 config、MCP tool、provider adapters 和 HTTP client 边界。
- `npm run build && npm run typecheck` 验证新 workspace 编译与类型。
- PR diff 中通过 examples 和 README 固定 managed/direct profile 的口径，避免把未实现的 hook/write/auto recall 描述成当前能力。
