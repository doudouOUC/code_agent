## 解决了什么问题

部分私有部署只需要“一个 interactive Qwen CLI 进程服务一个受信仓库”的外部上下文 provider：provider 凭据已经被限制到精确仓库语料，不需要 enterprise memory gateway 的多租户管理面。直接把 provider 或 provider selector 暴露给模型仍有风险：模型可能越过仓库边界、泄漏 credential 名称、把 provider 输出当可信指令、或在自动 recall 时携带 secrets。

## 最终实现

当前 open diff 新增 `integrations/external-context` workspace package，以 Qwen extension 形式接入，不改 Qwen Core API。配置层只接受 strict JSON，限制 64 KiB，校验真实 repository root、cwd containment、provider timeout 上限、credential env name 和 HTTPS/loopback HTTP endpoint；provider selector 不暴露给模型。

运行层提供三部分能力：`UserPromptSubmit` hook 做 fail-open 自动 recall，把经过敏感信息清洗的 prompt query 发送给 provider，并把结果作为 bounded untrusted JSON additional context 注入；MCP server 暴露只读 `context_search`，以及仅在配置允许时暴露 `context_remember`；provider adapter 支持 Generic HTTP Search V1 与 Mem0 Platform V3，统一 normalized context item。HTTP client 拒绝 redirects、oversized body、invalid JSON/UTF-8，并区分 provider timeout / response / transport error。render 层限制 item 数、单 item 内容和总输出长度，避免 provider 输出逃逸成指令。

## 关键代码路径

- `integrations/external-context/src/config.ts`, `config.test.ts`: strict config、repo root / cwd boundary、credential env 和 timeout 校验。
- `integrations/external-context/src/context.ts`, `context.test.ts`: auto recall query 清洗、manual query/content normalization、untrusted context rendering。
- `integrations/external-context/src/hook.ts`, `hook.test.ts`: `UserPromptSubmit` fail-open recall 与 remember tool ask boundary。
- `integrations/external-context/src/mcp.ts`, `mcp.test.ts`: `context_search` / optional `context_remember` MCP surface。
- `integrations/external-context/src/providers.ts`, `providers.test.ts`: Generic HTTP 与 Mem0 adapter、write result / unknown handling。
- `integrations/external-context/src/http-client.ts`: bounded POST JSON、URL validation、timeout/error taxonomy。
- `integrations/external-context/qwen-extension.json`, `hooks/hooks.json`: extension manifest 与 hook wiring。
