## 解决的问题

enterprise memory 的 runtime capture/recall 与 management consent/review/promotion/deletion 拥有不同调用方和权限。若共用一个网络平面，模型可调用 runtime credential 可能获得管理能力；同时 exact retry 需要在先授权后 replay 的前提下保持幂等。

## 当前实现方案

当前 open diff 新增 runtime gateway server、OIDC management server、OpenAPI contract 与 service entrypoint。Runtime plane 要求 mTLS、request-bound capability、当前 immutable runtime binding，并在任何 idempotency replay 前完成 tenant/principal/workspace/repository authorization；recall response 按 item count、per-item 和 total context budget 做 deterministic bounded rendering。

Management plane 独立验证 issuer、audience、tenant、subject、issued-at、expiration、最大 token lifetime，并按 personal/maintainer/tenant-policy scope 检查 consent、review、promotion、deletion authority。`openapi.yaml` 把 runtime 与 management routes 分开表达，`main.ts` 组合配置、store、memory service 与两个 server。

## 关键代码路径

- `integrations/enterprise-memory/src/gateway-server.ts`, `gateway-server.test.ts`: mTLS runtime plane、capability enforcement、authorization-before-replay、bounded recall rendering。
- `integrations/enterprise-memory/src/management-server.ts`, `management-server.test.ts`: OIDC token validation、scope-specific management authorization。
- `integrations/enterprise-memory/openapi.yaml`: API contract。
- `integrations/enterprise-memory/src/main.ts`: executable service entrypoint。
