## 解决的问题

合入 monitor notification batch drain，减少 stdout/running 事件导致的一条事件一次 LLM roundtrip。

## 最终实现方案

在 interactive CLI、nonInteractive session、nonInteractiveCli 和 ACP session 路径批量 drain 连续同类 notification；monitor callback 入队前过滤已 settled 的 stale running 事件；`MonitorRegistry.emitTerminalNotification` 复用 `TaskBase.notified` 防重复 terminal notification，并补 monitor registry、monitor tool、useGeminiStream、nonInteractiveCli focused tests。
