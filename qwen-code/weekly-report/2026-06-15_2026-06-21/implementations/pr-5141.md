## 实现目标

让一小类安全的单文件 `sed -i` 替换命令走编辑确认和 file history 记录，补 `/rewind` 对 shell 原地编辑的缺口。
PR body 只能辅助理解意图；本页的最终实现口径以 changed files、已提取的 patch 线索、测试和配置路径为准。

## 最终实现方案

- 实现摘要：新增 sed edit parser，shell 确认阶段读取目标文件并模拟替换生成文件 diff；执行前重新读取做 stale-content guard，写入前调用 `FileHistoryService.trackEdit()`，再通过 `FileSystemService.writeTextFile()` 落盘；不支持的 sed 形式继续走原 shell 路径，同时隐藏这类 shell-backed edit 的修改入口并补 core/CLI focused tests，已于 06-18 合入。
- 代码落地：新增或调整的实现符号包括 `SedEditInfo、ReadTextFileResponse、PreparedSedEdit、LEADING_ENV_ASSIGNMENT_RE、BACKSLASH_PLACEHOLDER、PLUS_PLACEHOLDER、QUESTION_PLACEHOLDER、PIPE_PLACEHOLDER`，这些符号把 PR 目标落到具体代码入口、类型、常量或服务实现上。
