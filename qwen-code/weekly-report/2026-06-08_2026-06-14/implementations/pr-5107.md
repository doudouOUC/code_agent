## 解决的问题

修复 OpenAI-compatible provider 复用或 replay `tool_call.id` 时引发的重复 tool result、payload 膨胀和 provider 校验错误。

## 最终实现方案

新增 `toolCallIdUtils`，同 turn 重复 id 只执行一次，跨 turn 复用 raw id 加 suffix；OpenAI parser/converter 出站前清理为一组 assistant call + 相邻 tool result，并给 core scheduler、non-interactive、AgentCore、ACP Session、speculation 补执行守卫。
