## 实现目标

给 `qwen serve` 增加按 session 的 prompt admission backpressure，避免单 session 无限堆积已接受 prompt。
PR body 只能辅助理解意图；本页的最终实现口径以 changed files、已提取的 patch 线索、测试和配置路径为准。

## 最终实现方案

- 实现摘要：bridge 维护默认 5 个 pending prompt 上限并同步拒绝；REST 返回 `503` + `Retry-After` + `prompt_queue_full`，ACP 映射结构化 JSON-RPC 错误，capabilities/CLI/SDK 增加 limit 配置和本地 reservation。
- 代码落地：新增或调整的实现符号包括 `DEFAULT_MAX_PENDING_PROMPTS_PER_SESSION、PromptQueueFullError、isNonNegativeIntegerOrInfinity、advertisedMaxPendingPromptsPerSession、MAX_DAEMON_BROWSER_BUNDLE_BYTES、normalizePendingPromptLimit、DaemonPendingPromptLimitError、NonBlockingPromptAccepted`，这些符号把 PR 目标落到具体代码入口、类型、常量或服务实现上。
