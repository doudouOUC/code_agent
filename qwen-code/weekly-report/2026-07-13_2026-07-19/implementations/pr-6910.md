## 解决的问题

#6911 给 daemon/SDK 增加了 workspace-qualified archived session export，但 Web Shell archived session row 仍没有 Export action。用户在 sidebar 里已经能看到 trusted primary/secondary archived rows，却只能手写 REST 或 SDK 调用导出，且 same-id archived rows 需要按 owning workspace 路由，不能复用 primary export。

## 最终实现方案

最终实现在 Web Shell sidebar 中新增 archived export gate。`canExportArchivedSessions` 只由 `workspace_archived_session_export` capability 决定；`getArchivedExportWorkspaceCwd()` 再确认 row 所属 workspace 已显示且 trusted，只有同时满足时 archived row actions 才显示 Export。untrusted archived workspace、缺 capability 或找不到 owning workspace 时不渲染导出入口。

`handleExportSession()` 对 active row 保持原有 `exportSession(sessionId, 'html')` primary 行为；对 archived row 则通过 `workspace.client.workspaceByCwd(workspaceCwd).exportArchivedSession(sessionId, { format:'html' })` 调 owning workspace client。exporting state 继续使用 session identity，结合 #6912 的 `(workspaceCwd, sessionId)` 行级身份，same-id rows 在不同 workspace 的下载和 disabled 状态互不影响。下载仍使用返回的 content、MIME type 和 filename，不改 active secondary read-only 边界。

## 关键代码路径

- `packages/web-shell/client/components/sidebar/WebShellSidebar.tsx`: capability gate、owning workspace trust check、archived export action 和 download flow。
- `packages/web-shell/client/components/sidebar/WebShellSidebar.workspace-removal.test.tsx`: archived export row actions、capability/trust hiding、same-id workspace isolation。
- `docs/users/qwen-serve.md`: 说明 Web Shell 在 trusted archived rows 上暴露 export。

## 验证方式

- Web Shell 测试覆盖：缺 `workspace_archived_session_export` 时隐藏 Export；untrusted archived workspace 隐藏 Export；trusted primary/secondary archived rows 走 owning workspace client；same session id 跨 workspace 时导出状态和下载内容隔离。
- 手工 smoke：连接支持 #6911 的 daemon，展开 Archived，确认 trusted archived row 有 Export，active secondary row 仍保持既有 read-only 行为。
