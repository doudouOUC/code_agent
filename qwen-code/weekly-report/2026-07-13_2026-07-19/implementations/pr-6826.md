## 解决的问题

multi-workspace daemon 中，singular `rewind` 和 `shell` route 仍按 primary runtime 处理。可信 secondary live session 因此可能从 primary bridge 读取 rewind snapshot、无法执行 rewind/shell，或者在 SDK 走 ACP transport 时绕开 strict REST mutation 鉴权。由于 rewind 会截断会话并恢复文件，shell 会执行本地命令，这些 route 必须按 live owner runtime 精确分发。

## 最终实现方案

PR 将 `GET /session/:id/rewind/snapshots` 改为 `withOwnerReadSession`，将 `POST /session/:id/rewind` 和 `POST /session/:id/shell` 改为 `withOwnerMutableSession`。route 先解析 session owner，再调用 `runtime.bridge.getRewindSnapshots()`、`runtime.bridge.rewindSession()` 或 `runtime.bridge.executeShellCommand()`。shell 完成日志与 rewind 日志补充 `workspaceId` / `workspaceCwd`，便于多 workspace daemon 的审计归因。

能力层新增两个条件 tag：`multi_workspace_session_rewind` 在 multi-workspace sessions 启用时广告，`multi_workspace_session_shell` 还要求 session shell flag/token 同时有效。`rewindFiles` 从旧的 truthy/falsy 宽松读取改成可选 boolean，非 boolean 返回 `400 invalid_rewind_files_flag`；省略或 `true` 恢复 tracked 文件，`false` 只回退对话。

SDK 层明确 rewind API 永远走 direct REST，即使 `DaemonClient` 配置了 ACP WebSocket/HTTP transport，也通过 authenticated REST 调用 owner-aware route，从而保留 strict mutation auth、client id 与 timeout 语义；shell 仍保持既有 transport 行为。daemon telemetry middleware 增加按 session id 查 owner runtime 的回调，使 legacy session route 的 workspace hash 不再只能落到 primary。

- `packages/cli/src/serve/routes/session.ts`: rewind snapshots、rewind、shell route owner-aware dispatch 与 `rewindFiles` 校验。
- `packages/cli/src/serve/capabilities.ts`: 新增 `multi_workspace_session_rewind` / `multi_workspace_session_shell` 条件能力。
- `packages/cli/src/serve/server.ts`, `server/telemetry.ts`: request telemetry 增加 session owner workspace resolver。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`: rewind snapshots / rewind 强制使用 REST transport，`fetch` fallback 可复用 transport 提供的 `restFetch`。
- `packages/sdk-typescript/src/daemon/*Transport.ts`: ACP/WS/auto-reconnect transport 暴露 REST fetch 兼容面。
- `packages/cli/src/serve/multi-workspace-sessions.test.ts`, `server.test.ts`: secondary owner dispatch、capabilities、invalid flag、SDK transport 行为覆盖。
