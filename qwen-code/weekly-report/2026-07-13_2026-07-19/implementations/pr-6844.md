## 解决的问题

legacy `GET /session/:id/export` 按设计绑定 primary workspace。multi-workspace daemon 中，trusted secondary workspace 的 active persisted session 已经可以被列出和分页读取，但完整 export 仍会回到 primary：secondary-only session 会 404；如果两个 workspace 有相同 session id，还可能导出 primary transcript，造成存储归属错误。

## 最终实现方案

PR 新增 `GET /workspaces/:workspace/session/:id/export?format=html|md|json|jsonl` 与独立 `workspace_session_export` capability。route 使用 workspace id 或 encoded absolute cwd 解析 selected runtime，并要求该 runtime trusted。解析和 trust 检查先于 session/format 校验；成功后只读取 selected workspace 的 active persisted JSONL。

实现上把 legacy export 抽成 `handleSessionExport()`，legacy `/session/:id/export` 仍传 `boundWorkspace`，plural route 传 selected `runtime.workspaceCwd` 和 `workspaceQualified:true`。两条 route 共用 `parseSessionExportFormat()`、`assertSessionLoadable()`、`exportSessionTranscript()`、filename sanitization、MIME type、`Cache-Control: no-store`、`X-Content-Type-Options: nosniff`、attachment header 和 archive coordinator shared lock。

plural route 不搜索其它 workspace、不 fallback primary、不解析 live owner、不启动 ACP、不 attach client、不加载 settings。unknown selector 返回 `workspace_mismatch`，untrusted 返回 `untrusted_workspace`，selected workspace 内找不到 session 返回 `session_not_found`；archived/archiving/conflict 继续复用既有 storage error shape。SDK 在 `WorkspaceDaemonClient.exportSession()` 上新增方法，强制 native REST，即使 parent client 使用 ACP transport，也保留 bearer token、client id、timeout、content type、attachment filename 和非 2xx error parsing。telemetry normalizer 增加 workspace export path，workspace hash 按 selected workspace 归因。
