## 解决的问题

multi-workspace daemon 已经把 session、workspace-qualified core REST、ACP transport、channel workers 和 runtime removal 分离到 selected runtime，但 Voice 仍停留在 primary workspace。secondary workspace 不能使用自己的 voice settings、runtime env 和 credentials 做转写；运行中的 secondary Voice stream 也不会被 workspace removal 识别，force removal 无法只关闭目标 workspace 的 Voice 工作。

## 最终实现方案

PR 新增 `workspace_qualified_voice` 条件能力，能力在 ACP HTTP 可用且 multi-workspace sessions 启用时广告。HTTP surface 增加 `GET/POST /workspaces/:workspace/voice` 与 `POST /workspaces/:workspace/voice/transcribe`，WebSocket surface 增加 workspace-qualified Voice stream upgrade。workspace selector 复用既有 plural route 规则：优先按 workspace id，再按 encoded absolute cwd；所有 mutation/transcription 都要求 selected runtime trusted，不 fallback primary。

route 层把 legacy Voice 逻辑拆成可复用 helper。selected runtime 提供 `boundWorkspace`、effective env、settings persistence scope、bridge 和 client id parser；settings 写入按 selected workspace scope 持久化，并在 partial persist 时广播已提交的 settings change。batch transcription 与 streaming transcription 使用 selected runtime 的 env/model 配置，legacy `/workspace/voice` 和 `/voice/stream` 仍绑定 primary。

并发和 lifecycle 由 `WorkspaceVoiceCoordinator` 管理。legacy 与 workspace-qualified REST/WS Voice 共用一个进程级 admission coordinator，保留总上限 8。admission lease 在真实 operation 开始前可随 HTTP finish/close 释放；operation 开始后由 transcription/stream settle 释放。workspace drain 后拒绝新 Voice work，返回 `workspace_draining` 或 `voice_capacity_exceeded`；non-force runtime removal 把 active Voice lease 计入 `activity.voiceSessions`，force removal/shutdown abort 目标 runtime 的 lease，并最多等待 5 秒收敛。

SDK 增加 workspace Voice helpers，让 `WorkspaceDaemonClient` 可以读取/更新 Voice settings、发起 batch transcription。Web Shell workspace sidebar 只新增 Voice activity 可见性，不改变 secondary workspace Voice UI 控件范围。telemetry path normalizer 识别 workspace-qualified Voice route，workspace hash 按 selected runtime 归因。
