## 解决的问题

persisted session 的 JSONL 通过 `parentUuid` 形成一棵树，但现有 resume/reconstruct 只会沿物理尾部恢复一条链。rewind、压缩、file-history checkpoint、attribution record 或并发写入都可能留下多个有效 sibling branch；只看物理尾会隐藏可恢复工作，也无法诊断 missing parent、cycle 或 duplicate parent conflict。

同时，custom-title、session-artifact write 这类 metadata-only terminal record 不应被误认为对话分支；而 rewind、compression、attribution、file-history checkpoint 等 system record 又有恢复意义，不能全部丢弃。

## 最终实现方案

最终实现新增只读 `inspectConversationBranches(records)`。它先按 `uuid`/`parentUuid` 建立 persisted transcript forest，并报告三类诊断：记录引用缺失 parent、parent 链形成 cycle、同一 UUID 出现 conflicting parent。分析过程不修改 transcript、不选择 active branch、不改变 resume/fork/daemon/ACP/CLI 行为。

branch 识别以“semantic leaf”为边界：真实 user/assistant/tool result 和恢复有意义的 system record 可成为分支尾；neutral metadata tail 会被折叠并去重，避免 custom title 或 artifact metadata 生成 phantom branch。每条 branch summary 包含 leaf、branch point、depth、updatedAt、用户/助手摘要、tool result 计数、与 rewind 的结构关系，以及 deterministic ordering。

rewind 后代和 rewind sibling 只作为结构关系分类：实现会识别从 rewind record 派生出的 descendant，也会识别和 rewind path 分叉的 sibling，但不会把 sibling 标记为 abandoned。后续写侧或 UI 可以基于该只读拓扑决定如何展示/恢复分支。
