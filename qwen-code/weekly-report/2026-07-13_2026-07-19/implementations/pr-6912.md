## 解决的问题

Web Shell 已能显示 registered secondary workspace 的 active/archived sessions，daemon 也已有 workspace-qualified archive routes，但 UI 层仍容易把不同 workspace 的同 id session 混在一起。current selection、busy state、export-in-flight、dedup、React key 和 post-operation reconciliation 如果只看 session id，会导致 archive/unarchive 操作或 UI 状态落到错误 workspace。secondary workspace 的 capability/trust gate 和 partial failure 显示也需要补齐。

## 最终实现方案

PR 将 Web Shell session row 的身份统一收敛为 `(workspaceCwd, sessionId)`。merged collection dedup、React keys、current selection、busy state、unread completion、export-in-flight state 都用该 identity，same-id primary/secondary rows 不再互相覆盖。workspace-qualified archive/unarchive response 中的 `errors[]` 会映射回对应 row 并展示；`alreadyArchived` / `alreadyActive` 继续视为 idempotent success。

能力边界保持保守：archive UI 需要 `session_archive`；secondary workspace 还需要 `workspace_qualified_rest_core` 且 runtime trusted。trusted secondary active row 只开放 Archive，pin/group/rename/export/delete 仍保持 load-only/read-only 边界。缺 capability 或 untrusted workspace 时不查询 archived catalog。每次 archive/unarchive settle 后，Web Shell 重新拉取 primary active/archived catalog 和 selected workspace active/archived catalog，避免 partial response 后前端状态漂移。

## 关键代码路径

- `packages/web-shell/client/components/sidebar/WebShellSidebar.tsx`: `(workspaceCwd, sessionId)` identity、archive/unarchive action gate、busy/export/current state 隔离、post-operation reconciliation。
- `packages/web-shell/client/components/sidebar/WebShellSidebar.workspace-removal.test.tsx`: secondary archive/unarchive 成功、partial failure、missing capability、untrusted workspace、same-id state isolation。
- `packages/cli/src/serve/multi-workspace-sessions.test.ts`: equal-id secondary archive/unarchive 不影响 primary file/bridge。
- `docs/design/web-shell-non-primary-session-archive-hardening.md`: Web Shell archive hardening contract。

## 验证方式

- Web Shell unit tests 覆盖 trusted secondary archive/unarchive、缺 capability 隐藏、untrusted hiding、idempotent alreadyArchived/alreadyActive、same-id current/busy/export state isolation。
- daemon regression 验证 same id 的 secondary session archive/unarchive 时 primary storage 和 bridge 不受影响。
- 手工 smoke：两个 workspace 各有同 id session 时，在 secondary row 上 archive/unarchive，确认 primary row 的状态、当前选中和 busy/export indicator 不变。
