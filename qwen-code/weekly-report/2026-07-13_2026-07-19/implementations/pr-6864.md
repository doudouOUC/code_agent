## 解决的问题

前台 shell timeout 以前表现为一个成功 `ToolResult` 里的普通文本。scheduler、ACP、stream-json、Anthropic converter、speculative UI 和 batch offload 都可能把已经被终止的命令当成功处理，并向模型发送 `response.output`。如果用户 cancel 晚于 timeout 到达，还可能覆盖 timeout 分类；PTY discovery 期间 abort 后也可能继续走向进程创建。

## 最终实现方案（open 当前观察）

当前 open diff 把前台 shell timeout 统一归类为 `ToolErrorType.EXECUTION_TIMEOUT`。Shell tool 返回结构化错误，同时在 model-facing 内容里保留超时摘要、部分 stdout/stderr 或明确的 no-output message，以及已有的 truncation pointer；operational error 只保留短摘要，供 hooks、telemetry、span 和顶层 `response.error` metadata 使用，避免把大量命令输出写入运维错误字段。

Core scheduler 在全局 tool execution timeout 与 shell 自身 timeout 之间区分 `schedulerTimeoutWon`，只有 timeout 先赢且不是后到 cancel 覆盖时才按 timeout failure 处理。timeout failure 使用 `convertToFunctionErrorResponse()` 写入 `functionResponse.response.error`，并保留 `resultDisplay` 给交互历史。普通 tool error 继续走既有错误路径。batch response size、active tool result offload 和 context extraction 都开始统计 `response.error` 文本，但不会把错误结果改成 success。

协议和 UI 适配也同步改成 error envelope：ACP Session 对 soft failure 使用 `convertToFunctionErrorResponse()`，tool execution span error 从 `tool_error` 细分 `tool_timeout`；non-interactive JSON 优先展示 model-facing error；speculative accepted tool result 看到 `response.error` 时显示失败；Anthropic converter 设置 `tool_result.is_error: true`；follow-up speculation 和 compaction input slimming 保留错误语义。Shell startup 在 signal 已 abort 或 PTY discovery 期间 abort 时停止，不再 spawn。

## 关键代码路径

- `packages/core/src/tools/shell.ts`: shell timeout result、startup abort/PTY abort 分类。
- `packages/core/src/services/shellExecutionService.ts`: abort reason 与 timeout/cancel/promotion refused 区分。
- `packages/core/src/core/coreToolScheduler.ts`: `convertToFunctionErrorResponse()`、timeout failure branch、error text offload/context size。
- `packages/cli/src/acp-integration/session/Session.ts`: ACP soft failure error envelope 与 `tool_timeout` span 分类。
- `packages/cli/src/nonInteractive/io/BaseJsonOutputAdapter.ts`: JSON 输出优先使用 responseParts error。
- `packages/cli/src/ui/AppContainer.tsx`: speculative tool result 按 `error` 字段标记失败。
- `packages/core/src/core/anthropicContentGenerator/converter.ts`: Anthropic `tool_result.is_error`。
