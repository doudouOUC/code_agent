## 解决的问题

#6798 和 #6826 已把多条 legacy route 改为 owner-aware，但仍有 4 条 session mutation 绑定 primary-only guard：继续中断 turn、修改输出语言、添加 artifact、删除 artifact。可信 secondary live session 因此无法完成这些操作，常见表现是返回 `non_primary_session_route_not_supported`，而不是像 primary session 一样工作。

## 最终实现方案（open 当前观察）

当前 open diff 延续同一模式，把 `POST /session/:id/continue`、`POST /session/:id/language`、`POST /session/:id/artifacts` 和 `DELETE /session/:id/artifacts/:artifactId` 改为 `withOwnerMutableSession`。route 在参数校验、client id 解析和错误映射后调用 owning `runtime.bridge`，不再直接调用 primary `bridge`。

continue 仍走 strict mutation auth，保留可选 client identity，并在 route 层生成新的 `promptId` 传给 bridge；language route 保留既有非 strict auth posture，继续传递 `language`、`syncOutputLanguage` 和可选 client id；artifact add/delete 保留 required client id、payload/id 校验、artifact validation error 映射和必须针对 live session 的约束。unknown owner 返回 `session_not_found`，untrusted owner 返回 `untrusted_workspace`，ambiguous owner 返回 `ambiguous_session_owner`，均不 fallback primary。
