## 解决的问题

extension artifact 是 user-level 资源，多个 workspace runtime 会共享同一个 `QWEN_HOME/extensions`。旧的 `/workspace/extensions` 兼容面把 artifact 安装和 workspace 启用状态混在 primary workspace 里，既不能表达「全局 artifact + workspace activation policy」，也无法安全处理并发 install/update/uninstall、runtime refresh 失败、外部 daemon 进程写入和敏感 settings commit。多 workspace daemon 需要一个明确的事务边界：artifact/state 是全局单 writer，activation 可以按 workspace 覆盖，runtime refresh 失败要作为 warning 告诉用户但不能回滚已提交状态。

## 最终实现方案（open 当前观察）

当前 open diff 新增 `extension_management_v2` additive capability，并保留 `workspace_extensions` 与 `/workspace/extensions/*` 作为 primary compatibility adapter。V2 的核心是 `ExtensionStore`：它管理 user-level extension artifact、`extension-store/state.json`、单调 `generation`、activation policy、legacy enablement projection、staging/trash/rollback/transaction journal 和 recovery。安装或更新先在最终 artifact 目录外 prepare，commit 时串行写 artifact 与 state；journal phase 从 `prepared` 到 `artifact_swapped` 再到 `state_committed`，崩溃恢复在 state commit 前 rollback，在 commit 后完成 cleanup。

daemon 侧 `ExtensionsController` 提供 operation status 和排队控制。准备阶段用 FIFO queue 限制最多 2 个并发 task；durable commit lane 是单通道，按 preparation 完成顺序串行写 artifact/state，并在这些写入持久化后释放 lane，让 runtime refresh 与 cleanup 不阻塞下一个 durable commit。未完成 operation 超过 10 个时返回 `429 extension_queue_full`。每个 mutation 返回 `202 operationId`，客户端通过 `/extensions/operations/:operationId` 轮询，结果可能是 `succeeded`、`succeeded_with_warnings` 或 `failed`。

全局 route `/extensions/*` 负责 catalog、install、check updates、update、uninstall、default activation 和 operation status；workspace route `/workspaces/:workspace/extensions/*` 负责 selected workspace 的 effective projection、activation override、clear override 与 runtime refresh。全局 artifact/default 变更会 reconcile 所有 hosted runtimes；workspace activation 只 reconcile 目标 runtime。reconciliation 刷新 extension/skill cache、extension tools、hierarchical memory、active chat system instructions 和 available commands。post-commit refresh、settings sync 或 cleanup 失败不会回滚 committed generation，而是作为 warning 透出；settings/cleanup warning 使用 `updated with warnings`，reload/runtime refresh 失败保留 restart 引导。

敏感 settings 进入 artifact transaction 但不直接写入 artifact。prepare 阶段写一个 revisioned secret bundle 到选定 secure-storage backend，staged artifact 只放 backend/revision selector；commit 原子激活完整 bundle，取消会删除未选中 bundle，成功替换会清理旧 bundle，legacy per-key sync 在 durable lane 外继续。这样崩溃最多留下不可达 orphan bundle，不会出现新 artifact 搭配旧或半写入 secret。

安全与兼容方面，GitHub/npm 下载使用 `network-policy.ts` 的 public policy：强制 HTTPS、拒绝 URL credential、DNS 解析到 private/link-local/loopback/reserved/multicast 等地址时拒绝，并把 lookup pin 传给请求/curl resolve，降低 DNS rebinding 风险。extension file watcher 观察 store generation 和 linked source，外部 commit 后其它 daemon 能收敛。legacy install/link/enable/disable/uninstall/update/TUI action 会展示 committed warning，不再只报告成功。

daemon shutdown 也被补强：runtime-owned cleanup 针对实际 runtime app 执行，包括 shutdown 开始后才完成构建的 deferred app；scheduled-task keepalive、workspace Git state、sub-session launcher、extension reconciliation、device flow、ACP、rate limiting、event-loop monitor、metrics sampler 等都幂等 dispose，单个 cleanup callback 抛错不会阻断其它资源释放。

## 关键代码路径

- `packages/core/src/extension/extension-store.ts`: V2 store、generation、activation policy、journal/recovery、legacy projection。
- `packages/core/src/extension/extensionSettings.ts`: revisioned secret bundle、selector、legacy per-key sync 与 cleanup。
- `packages/core/src/extension/extensionManager.ts`: prepare/commit/cancel、settings activation、post-commit warning 与 cleanup。
- `packages/core/src/extension/network-policy.ts`: public extension download network policy、DNS/IP block、pinned lookup/curl resolve。
- `packages/cli/src/serve/extension-operation-scheduler.ts`: FIFO preparation/commit queue。
- `packages/cli/src/serve/routes/workspace-extensions-controller.ts`: operation lifecycle、preparation/commit/reconciliation、warning 归集与 history pruning。
- `packages/cli/src/serve/routes/workspace-extensions.ts`: global `/extensions/*`、workspace `/workspaces/:workspace/extensions/*`、legacy adapter 与 operation polling。
- `packages/cli/src/config/extension-file-watcher.ts`: store generation / linked source watcher。
- `packages/cli/src/commands/extensions/*.ts`, `packages/cli/src/ui/components/extensions/*`: legacy CLI/TUI action 的 warning 展示和 V2 surface。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`, `types.ts`: extension catalog/mutation/polling/workspace projection SDK helper。
- `packages/web-shell/client/App.tsx`: Web Shell 接入 extension management surface。
- `packages/cli/src/serve/run-qwen-serve.ts`: runtime app cleanup、deferred runtime late disposal、extension reconciler dispose。
