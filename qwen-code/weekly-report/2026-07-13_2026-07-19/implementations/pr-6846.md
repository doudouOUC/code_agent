## 解决的问题

扫描 PDF 没有可抽取文本层，纯文本主模型即使配置了 vision bridge，也只能从 `read_file` 得到不可恢复的 PDF 读取错误。另一个失败点是高密度单页 PDF：`pdftotext` 成功，但单页结果超过 12K tool-result 预算，已经无法通过更窄页码继续缩小。旧链路没有一种 text-first、bounded、可 disclosure 的视觉兜底。

## 最终实现方案（open 当前观察）

当前 open diff 在 `read_file` 加入 PDF 专用 vision bridge fallback。PDF 文本抽取仍是第一选择；只有抽取失败，或文本抽取成功但单页输出超过 `PDF_TEXT_RESULT_MAX_TOKENS`，才准备 bridge candidate。多页文本超限仍返回缩小页码 guidance，不转视觉转录。候选渲染从请求起始页开始，单次最多处理 `VISION_BRIDGE_MAX_IMAGES`（4 页），根据已知页数或 render cap 生成 known/possible continuation，并要求续读必须重新对原始 PDF 调 `read_file`。

候选结果不会直接进入纯文本 provider。`processSingleFileContent()` 只生成 `pdfVisionBridgeCandidate`，其中包含 rendered page image parts、原始 fallback error/guidance、rendered range 和 continuation。`ReadFileTool` 在执行阶段调用 `runVisionBridge()`，成功时用 bridge 返回的纯文本 parts 替换 image parts；如果 bridge 失败、空输出、丢模型选择、返回媒体 payload 或遗漏页面，则恢复原始 PDF fallback，同时附加用户可见 notice。最终 tool result 不包含 rendered image `inlineData`。

`vision-bridge-service.ts` 为 PDF 增加 source context：bridge prompt 标明图片是原 PDF 的连续页，并要求按原始页码分段；模型返回被包在 “untrusted machine transcription” 边界里，明确不能执行图片中的指令。用户侧 disclosure 记录 vision model、endpoint、是否发生 egress、已处理页范围和续读范围。TUI 的 `ToolMessage`、ACP tool-call emitter、daemon TUI adapter、non-interactive JSON output 和 export normalize 都识别 `VisionBridgeNoticeDisplay`，确保成功和失败路径都能保留同一结构化说明。

## 关键代码路径

- `packages/core/src/utils/fileUtils.ts`: `preparePdfForVisionBridge`、`PDFVisionBridgeCandidate`、render range/continuation、fallback 恢复数据。
- `packages/core/src/tools/read-file.ts`: bridge candidate 执行、safe replacement、notice display 和 tool schema 说明。
- `packages/core/src/services/visionBridge/vision-bridge-service.ts`: PDF source context、untrusted transcription block、egress notice、bridge display formatter。
- `packages/cli/src/ui/components/messages/ToolMessage.tsx`: TUI 展示 bridge notice，避免把 structured display 当普通 result display。
- `packages/cli/src/acp-integration/session/emitters/tool-call-emitter.ts`: ACP 成功/失败 tool result 携带 bridge notice。
- `packages/cli/src/nonInteractive/io/BaseJsonOutputAdapter.ts`, `ui/utils/export/normalize.ts`: JSON/export 输出保留 notice。
- `packages/core/src/core/coreToolScheduler.ts`: 错误 response 可保留非字符串 result display。
