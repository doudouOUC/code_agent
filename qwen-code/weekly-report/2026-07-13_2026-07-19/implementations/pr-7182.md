## 解决的问题

P0-A startup profile 显示 Gemini 和 ACP module load 占 child startup P50 的 67.3%。import graph 进一步发现无头 ACP child 的静态闭包会编译 Ink、React、React Reconciler 和 Yoga；这些模块只服务 TUI render，但 ACP child 不渲染 UI。继续让 API error/suggestion contract 经过 React/TUI 模块，会把低频交互依赖带进每次 ACP 冷启动。

## 最终实现方案

最终实现把纯数据契约从 TUI runtime 中拆出。`classifyApiError` 从 `useGeminiStream.ts` 移到 `packages/cli/src/utils/classify-api-error.ts`，ACP `Session.ts` 可直接复用而不加载 React hook。Suggestion 类型、最大展示数和映射逻辑从 `SuggestionsDisplay.tsx` 移到 `packages/cli/src/ui/utils/suggestions.ts`，UI component 只负责渲染。

低频交互命令的 React-only 依赖改为按动作执行时动态导入：`/init` 非空 QWEN.md 覆盖确认、`/approval-mode auto` 和 `/history expand-now` 保持原 UI 结果，但不再静态进入 ACP startup closure。React types 通过 type-only import 被擦除，不再制造 runtime edge。

bundle guard 新增 ACP static closure 检查：`scripts/check-serve-fast-path-bundle.js` 从 metafile 找到 ACP runtime root，禁止 Ink、React、React Reconciler、Yoga package inputs 进入 static closure，同时允许有意的 dynamic imports。设计文档记录同 SHA 2C4G 证据：60 组 alternating cold pairs 中 ACP import P50 从 115.06ms 降到 52.00ms，`channel.initialize` P50 降 62.64ms，process-to-first-Session P50 降 66.85ms，preheated/P95/RSS gate 无回退。

## 关键代码路径

- `packages/cli/src/utils/classify-api-error.ts`: API error classifier 从 React hook 解耦。
- `packages/cli/src/ui/utils/suggestions.ts`: suggestion data contract、display limit 与映射 helper。
- `packages/cli/src/ui/components/SuggestionsDisplay.tsx`, `useGeminiStream.ts`: TUI component/hook 改为消费纯工具模块。
- `packages/cli/src/ui/commands/initCommand.ts`, `approvalModeCommand.ts`, `historyCommand.ts`: 低频 React UI action 改为动态导入。
- `packages/cli/src/acp-integration/session/Session.ts`: ACP startup path 只依赖纯 classifier。
- `scripts/check-serve-fast-path-bundle.js`: ACP static closure forbidden package guard。
- `docs/design/acp-channel-initialize-profiling.md`: P0-A selected optimization、rejected alternatives 和 2C4G evidence。
