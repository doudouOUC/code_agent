## 解决的问题

daemon-managed channel worker 中 adapter `connect()` rejection 只写 worker 日志，父进程只知道 ready 或最终 `No channels connected.`。动态 `qwen channel set/reload`、SDK 和 HTTP API 都拿不到 provider 的实际失败 message/code，也无法区分 partial connect 与 all-fail rollback。用户排查 channel 启动失败只能翻日志，而且容易丢失 workspace/channel attribution。

## 最终实现方案

#6950 增加 channel startup failure IPC 协议。worker 对每个 adapter `connect()` catch 发送 `channel_startup_failure`，其中包含 bounded/redacted channel、phase（当前 `connect`）、optional code 和 message，并等待父进程 `channel_startup_report_ack` 后才尝试下一个 adapter。send callback 只代表 Node 接收消息，ACK 才代表 supervisor 已处理，避免 worker 同步退出时丢 failure。

父进程验证 IPC shape、清洗控制字符/不可见字符、精确 redact daemon token 和 sensitive env，再套 generic credential redaction，并按 Unicode code point 截断。每次 startup 最多传 64 条 failure；第 65 条生成一次 `startupFailuresTruncated` marker，后续只留 stderr。malformed、过长、out-of-order 或无法 ACK 的 startup protocol message 会 fail bounded startup 并终止 child；普通 unknown IPC message 保持原行为。

只要至少一个 adapter connect 成功，worker 仍 ready，snapshot 中暴露 `startupFailures[]` 和 truncation flag，dynamic enable 返回 success with partial state。若 dynamic enable/replace/reload 的所有 adapter 都失败，请求返回 `502 channel_worker_start_failed`，body 带 workspace-annotated attempted failures、truncation flag、rollback 后 current `state`，失败 attempt 不写入后续 GET snapshot。daemon boot-time all-failure 仍 fail-fast，因为 listener 不会保持可查询。

CLI `qwen channel set/status/reload` 使用 shared formatter 展示 startup failures；SDK type surface 增加 `ChannelStartupFailure`、`ChannelStartupAttemptFailure` 和 snapshot fields。failure object 在 worker/supervisor/API/SDK 边界都 clone，避免调用者修改 supervisor state。

## 关键代码路径

- `packages/cli/src/commands/channel/daemon-worker.ts`: child connect failure report、ACK wait、truncation marker。
- `packages/cli/src/serve/channel-worker-startup-ipc.ts`: IPC validation、redaction、ACK protocol。
- `packages/cli/src/serve/channel-worker-supervisor.ts`: startup failure storage、partial ready、all-fail `ChannelWorkerStartupError`。
- `packages/cli/src/serve/channel-worker-group.ts`, `channel-worker-manager.ts`: workspace annotation、rollback error propagation。
- `packages/cli/src/serve/routes/workspace-channel-control.ts`: `502 channel_worker_start_failed` response with attempted failures。
- `packages/cli/src/commands/channel/startup-failure-format.ts`: CLI formatting and caps。
- `packages/sdk-typescript/src/daemon/types.ts`: public snapshot/attempt failure types。

## 验证方式

- channel worker/supervisor unit tests 覆盖 partial/all failure、ACK ordering、abort/timeout、malformed IPC、ACK failure、64/65 truncation、generation reset、deep copies、redaction 和 rollback propagation。
- integration `qwen-serve-channel-workers.test.ts` 使用本地关闭端口制造 deterministic `ECONNREFUSED`，验证 dynamic control response 与 status display。
- CLI tests 覆盖 `qwen channel set/reload/status` failure formatting。
- SDK tests 覆盖 daemon public type exports。
- 手工 smoke：配置两个 channels，一个 connect 成功、一个拒绝连接，确认 status 显示失败原因；再配置全失败 dynamic set，确认 502 response 带 attempted failures 且旧 worker state 回滚。
