## 解决的问题

`exit_plan_mode` 过去把用户审批和 mode transition 混在 confirmation callback 里。AUTO/YOLO、permission-manager allow rule、permission allow hook 或 sibling auto-approval 可能满足 `ask` 决策，而没有真实 host/user response；LLM Plan Approval Gate 还允许模型用 `ask_user_question` metadata 自行尝试审批。callback 在 hooks/execution 完成前改 mode，后续失败时 ACP mode notification 也可能误报。

## 最终实现方案

最终实现为工具 invocation 增加 `requiresUserInteraction()`。它不是新的 permission level：intrinsic deny 和 PM deny 仍然优先，但 allow rules、automatic approval modes、AUTO_EDIT、YOLO、permission allow hooks 和 sibling auto-approval 不能满足该要求。main-session `exit_plan_mode` 声明 requires user interaction；plan-required teammate 仍保留 leader approval path；普通 subagent 继续用既有 lifecycle-tool rejection。

`exit_plan_mode` confirmation callback 只记录四种 decision：restore pre-plan mode、switch auto-edit、switch default、cancel。它冻结 plan text、pre-plan mode 和当前 approval-mode revision，但不改 mode。真正的 `execute()` 会检查 approval 已存在、signal 未 abort、session 仍在 Plan mode、revision 未变化，然后同步执行 mode transition；stale、re-entered 和 concurrent exit 都 fail closed。plan persistence 只在 transition 成功后 best-effort 执行，不回滚已成功的 mode change。

`Config` 新增 monotonic approval-mode revision，只有实际 mode change 才递增；approval-mode override 有独立 revision，子 agent override 不污染 parent。`enteredByModel` setter 参数暂时保留但忽略，model origin 不再影响审批。旧 LLM Plan Approval Gate、`plan-gate/*` state/review agents 和 AskUserQuestion metadata coupling 删除；`resolutionSummary` 只保留 deprecated TypeScript source compatibility，不再进 runtime schema。ACP 在 permission pending、confirmation/hooks/execution/transition 失败时不发 mode update，只有成功 mode change 后用 Config 当前 mode 发送一次 update。
