## 解决的问题

现有 startup profiler 能拆 CLI 进程启动阶段，但 #6312 后续优化需要定位 `GeminiClient.startChat()` 内部剩余成本。没有 session-start 级别的 stage timing 时，无法判断下一步应该优化 system instruction 构造、initial history、tool syncing、SessionStart hook 还是其它阶段。

## 最终实现

该 PR 新增 core 内部 opt-in profiler：`packages/core/src/core/session-start-profiler.ts` 在 `QWEN_CODE_PROFILE_SESSION_START=1` 时创建 profile，写入 `Storage.getRuntimeBaseDir()/session-start-perf/session-start-YYYY-MM-DD.jsonl`。关闭时不写文件，也不读高精度时钟。

`GeminiClient.startChat()` 在关键阶段调用 `profiler.time()` / `timeSync()`：tool registry warm、resume deferred tool reveal、deferred reminder resolution、initial chat history、skill/agent reminder seed、system instruction、`GeminiChat` construction、orphan tool-use repair、SessionStart hook、SessionStart context apply 和 `setTools()`。`finish()` 在成功和失败路径都会落 bounded record，记录静态 stage 名、source、success、总耗时、阶段耗时和小型聚合计数，不写 prompt、path、session id、hook output、model response 或 tool name。

`.qwen/design/2026-07-06-session-start-profiler.md` 记录安全边界和非目标：该 PR 只加测量，不优化 `initialize()`，也不改 protocol、SDK、config、telemetry schema 或 CLI flag。

## 关键代码路径

- `packages/core/src/core/session-start-profiler.ts`: opt-in JSONL profiler、stage timing、best-effort write 和敏感信息边界。
- `packages/core/src/core/client.ts`: `GeminiClient.startChat()` stage instrumentation 与 success/failure finalize。
- `packages/core/src/core/session-start-profiler.test.ts`: profiler 开关、输出形状、写失败吞掉、错误保留。
- `packages/core/src/core/client.test.ts`: startChat source、stage 顺序、成功/失败 finalize 和 bounded counts。
- `.qwen/design/2026-07-06-session-start-profiler.md`: 设计与安全边界说明。
