## 解决的问题

`qwen serve --channel` 的 channel worker selection 仍主要由启动参数决定。daemon 启动时没有带 `--channel`，后续就不能在不重启 daemon 的情况下启用 channel worker；已经启用的 worker 也缺少运行时替换、查询和停止 selection 的统一控制面。#6598 只解决了“已有 selection 的 settings reload”，#6635 只解决了“selection 下的 multi-workspace worker grouping”，还没有 runtime selection lifecycle。

## 最终实现方案

PR 新增 `ChannelWorkerManager`，把 worker selection 从 boot-only 配置提升为 daemon runtime resource。manager 持有当前 committed selection、channel worker group snapshot、PID file/webhook routing 状态，并串行化 set/reload/stop 这类 lifecycle mutation，避免并发请求交错 fork/kill worker。

对外新增条件能力 `channel_control`。能力存在时，客户端可以通过 `GET /workspace/channel` 查询当前 selection 和 worker 状态；`PUT /workspace/channel` 设置或替换 selection；`POST /workspace/channel/reload` 复用同一 reconcile path 重读 settings；`DELETE /workspace/channel` 停止并清空 runtime selection。runtime selection 是临时控制态：`PUT` 不写 settings，也不改 boot options；daemon 重启后仍回到 `qwen serve --channel` 启动参数指定的 selection，或在未传该参数时保持 disabled。`channel_reload` 仍表示 manager 已 wire reload deps；`channel_control` 表示 runtime selection resource 可用。

manager 复用 #6635 的 workspace-scoped `ChannelWorkerGroup.reconcile()`：替换 selection 时保留未变化 workspace worker，只对新增/删除/变化的 group 做启动或停止；如果新 selection 启动失败，会回滚到旧 selection 和旧 PID file/webhook routing 状态。PID file 的 committed state 与 worker group 切换保持原子语义，避免外部 channel webhook 路由到半更新状态。每个 worker callback 带 generation，替换前 worker 的 late ready/exit 只记录日志，不会覆盖当前 generation；daemon drain/shutdown 会让 set/reload/stop 稳定返回 `daemon_draining`。

CLI 层扩展 `qwen channel`：`set` 可把 selection 写到 daemon runtime，`status` 查询 daemon 当前 selection/worker，`stop` 删除 runtime selection；SDK 增加对应 channel control helper。worker shutdown 也被加固：未确认 child exit 前不释放 PID lease，避免 stale exit races 下重复 worker 同时存活。

## 关键代码路径

- `packages/cli/src/serve/channel-worker-manager.ts`: runtime channel selection manager、串行 lifecycle、rollback 和 snapshot。
- `packages/cli/src/serve/channel-worker-group.ts`: worker group reconcile，保留 unchanged workspace workers，并用 generation 忽略 stale callbacks。
- `packages/cli/src/serve/routes/workspace-channel-control.ts`: channel status/set/reload/stop HTTP routes。
- `packages/cli/src/serve/run-qwen-serve.ts`: manager wiring、capability/status deps 注入。
- `packages/cli/src/serve/server.ts`: route 注册与 capability toggles。
- `packages/acp-bridge/src/channel-control-timeouts.ts`: startup/stop/reload timeout 常量，经 package export 供 CLI/SDK 内部复用。
- `packages/cli/src/commands/channel/set.ts`, `status.ts`, `stop.ts`: daemon-aware CLI control commands。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`, `types.ts`: SDK channel control methods/types。
- `integration-tests/cli/qwen-serve-channel-workers.test.ts`: runtime control E2E 覆盖。
