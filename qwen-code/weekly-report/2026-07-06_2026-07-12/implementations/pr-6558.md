## 解决的问题

#6540 已允许 trusted non-primary workspace 通过显式 `cwd` load/resume persisted session，但 session list 仍只看 live bridge summary。这样 secondary workspace 里已经落盘的 active historical sessions 无法从 workspace session routes 被发现，用户只能恢复已知 session id，不能浏览该 workspace 的历史会话。

## 最终实现

该 PR 让 workspace session list 在 trusted non-primary workspace 上支持 active persisted recent view。路由先保留 Phase 2b 的边界：non-primary workspace 仍拒绝 archived、organized view 和 group 查询，只支持 active recent list；unknown selector 返回 `workspace_mismatch`，untrusted workspace 返回 `untrusted_workspace`。

实现上，`/workspace/:workspace/sessions` 和 `/workspaces/:workspace/sessions` 在选中 runtime 后先探测 `new SessionService(workspaceCwd).listSessions({ archiveState:'active', size:1 })`。如果该 workspace 有 active persisted sessions，走现有 `listWorkspaceSessionsForResponse()`，把 persisted list 与该 runtime 的 live summary 合并并去重；如果探测失败或没有 persisted sessions，则回退到 live-only listing，保持 Phase 2a 行为。

分页边界也做了修正：persisted-backed list 使用 numeric cursor；live-only list 继续使用 live cursor。若第一页是 live-only，后续请求即使中途出现新的 persisted session，也不会把 cursor 语义切到 persisted list，避免同一分页序列中重复或跳过 live session。
