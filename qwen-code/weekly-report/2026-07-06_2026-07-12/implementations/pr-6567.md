## 解决的问题

multi-workspace daemon 已经能把 session live loop 和 trusted secondary persisted session list/load/resume 分发到对应 runtime，但核心 REST 面仍大量绑定 primary workspace。客户端若要读写 secondary workspace 的文件、查看 settings/MCP/memory/agents，或管理该 workspace 的 sessions/group 状态，只能绕回 primary-only 路由，容易形成半迁移、半误路由的 API 面。

## 最终实现

该 PR 增加 Phase 3 workspace-qualified core REST contract：所有新增路由挂在 `/workspaces/:workspace/...`，selector 先按 workspace id 精确匹配，再把 URL-decoded 值按 portable absolute path 解析，支持 POSIX、Windows drive 和 UNC 风格路径。解析失败和未知 workspace 都返回 `workspace_mismatch`；registered but untrusted workspace 除文件读和 trust/status 这类现有策略允许的面外返回 `untrusted_workspace`。

路由层新增共享 helper `workspace-route-runtime.ts`，把 selector decode、absolute path 判断、canonical cwd lookup、trusted runtime gate 和 typed mismatch response 收敛到一处。`server.ts:createServeApp()` 挂载 plural core routes，同时 legacy `/workspace/...` 与 workspace-less session routes 保持 primary workspace 语义不变。

新增 plural routes 覆盖文件读写、stat/list/glob、workspace status/preflight/env/providers/skills/hooks、settings、permissions、trust、lifecycle reload/init、MCP restart/manage、tool toggles、memory、project-agent CRUD，以及 persisted session organization、archive/delete/group/batch 操作。session list 继续继承 #6558 的 trust 与 active/recent 边界。

SDK TS 新增 `workspaceById()` / `workspaceByCwd()`，返回 `WorkspaceDaemonClient`。该 client 把 workspace selector 封装进 `/workspaces/:workspace` 请求，并提供 file/status/settings/agents/memory/sessions/MCP/lifecycle 等 helper；bundle size guard 相应从 139 KiB 调到 150 KiB。daemon capability 新增 `workspace_qualified_rest_core`，并随 workspace settings/persist route deps 一起条件广告，多 workspace 客户端可用它 feature-detect。

- `packages/cli/src/serve/server.ts`: plural workspace core route 挂载。
- `packages/cli/src/serve/routes/workspace-*.ts`: file read/write、status、settings、permissions、trust、tools、MCP、lifecycle routes 的 workspace-qualified 版本。
- `packages/cli/src/serve/workspace-agents.ts`、`workspace-memory.ts`: workspace-qualified agents/memory CRUD。
- `packages/cli/src/serve/routes/session.ts`: plural workspace session organization/archive/delete/group routes。
- `packages/cli/src/serve/server/telemetry.ts`: plural workspace routes 归一化为稳定 telemetry route label。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`: `workspaceById()` / `workspaceByCwd()` 和 `WorkspaceDaemonClient`。
- `packages/cli/src/serve/workspace-qualified-rest.test.ts`: selector、trust、file/status/agents/memory route 行为覆盖。
