## 解决的问题

Phase 2a 后续要创建 non-primary workspace runtime，但当前 serve/acp-bridge 多处仍直接读父进程 `process.env`。这会让后续 workspace-local `.env` / settings env 隔离变得不可靠，也会让 settings reload 后的 runtime env 状态难以统一刷新。另一个缺口是 daemon-wide session admission：未来多个 runtime 并发创建 session 时，如果只看单 bridge `maxSessions`，可能在 session 注册前超卖总量。

## 最终实现

当前 open diff 新增 runtime-local env snapshot。`packages/cli/src/config/environment.ts` 抽出 `buildRuntimeEnvironment(settings, startDir, baseEnv)`，从 daemon boot 时的 base env、workspace `.env`、settings env 和 Cloud Shell 默认值合成 immutable `effectiveEnv`，同时返回 overlay keys、env file paths 和 readFailed 标记。生产 `runQwenServe()` 在 runtime boot 时构造 mutable `runtimeEffectiveEnv`，并在 daemon env reload 成功后重建 snapshot、原地替换该 env source；失败时保留上一份 env 并记录 fallback reason。

ACP child spawn 改为使用显式 `sourceEnv`。`createSpawnChannelFactory()` / `spawnChannel()` 不再在 spawn 时隐式读 live parent process env；status provider、workspace providers status、voice config、A2UI stdio transport、model auth type detection 等低成本 workspace-scoped 消费点都接收注入 env。`process-env-guard.test.ts` 为 serve/acp-bridge workspace-scoped 路径加 guard，防止后续又直接读 `process.env`。

总量 admission 通过 `BridgeOptions.freshSessionAdmission` 接入 bridge。`createTotalSessionAdmissionController()` 在 `spawn`、`load`、`resume`、`branch` 等 fresh session seam 上同步 reserve；existing-session attach 不调用 admission。命中 `--max-total-sessions` 时抛 `TotalSessionLimitExceededError(scope:'total')`，REST error response 复用 `session_limit_exceeded` 形状并加 `scope:"total"`，`Retry-After: 5` 保持与 workspace session cap 一致。reservation 在成功注册、失败或 branch restore 清理后 release，branch 内部 load 会设置 `skipFreshSessionAdmission`，避免同一次 branch 双重扣减。

`/daemon/status` 只做 additive 扩展：`limits.maxTotalSessions` 暴露配置，`runtime.sessions.admissionInFlight` 暴露当前 reserve 中数量，并在总量接近上限时生成 `total_session_capacity_high` warning。`/capabilities` 不新增 `workspaces[]`，也不发布 `multi_workspace_sessions`。

- `packages/cli/src/config/environment.ts`: `buildRuntimeEnvironment()`、env file parse 复用、Cloud Shell env 注入到目标 env。
- `packages/cli/src/serve/run-qwen-serve.ts`: runtime env snapshot 构建/刷新、`sourceEnv` 传给 ACP child spawn、status/provider env injection、total admission controller 装配。
- `packages/acp-bridge/src/bridgeOptions.ts`: `BridgeFreshSessionAdmissionContext` / `BridgeFreshSessionReservation` seam。
- `packages/acp-bridge/src/bridge.ts`: spawn/load/resume/branch 前 reserve，finally release，attach bypass。
- `packages/cli/src/serve/total-session-admission.ts`: daemon-wide fresh-session cap、in-flight counter、reservation release。
- `packages/cli/src/serve/server/error-response.ts`: `TotalSessionLimitExceededError` 映射到 `503 session_limit_exceeded` + `scope:"total"`。
- `packages/cli/src/serve/daemon-status.ts`: `limits.maxTotalSessions`、`runtime.sessions.admissionInFlight`、`total_session_capacity_high`。
- `packages/cli/src/serve/routes/a2ui-action.ts`、`voice/resolve-voice-config.ts`、`workspace-providers-status.ts`、`utils/modelConfigUtils.ts`: workspace-scoped env injection 消费点。
