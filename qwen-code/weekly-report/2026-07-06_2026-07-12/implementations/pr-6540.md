## 解决的问题

#6511 的 Phase 2a closed loop 已能把核心 live session 操作路由到 owning runtime，但 owner lookup 仍依赖扫描所有 runtime bridge；persisted load/resume 也仍是 primary-only。后续 multi-workspace session 需要一个更确定、低成本的 owner seam，同时在 missing、stale、ambiguous owner 场景继续 fail closed。

## 当前实现方案

当前 open diff 在 `WorkspaceRegistry` 中加入 `WorkspaceSessionOwnerIndex`。bridge 生命周期通过 `sessionLifecycle` callback 把 registered/removed 事件写入 index，覆盖 spawn、load/resume、channel exit、close、kill 和 daemon shutdown 等会话所有权变化。`resolveLiveSessionOwner(sessionId)` 先查 index，再用 indexed runtime 的 `getSessionSummary()` 校验；如果 entry stale 就清理并回退到 bridge scan，scan 命中后回填 index。scan 中任何 bridge lookup 抛非 not-found 错误时，不缓存部分结果，避免污染 owner index。

sessions-only surface 扩展到 trusted non-primary persisted load/resume。`POST /session/:id/load` / `resume` 接受显式 `cwd`，unknown cwd 返回 `workspace_mismatch`，untrusted registered workspace 返回 `untrusted_workspace`；如果同一 session id 已在另一个 runtime live 或 restore 中，则返回 `409 session_workspace_conflict`，避免一个 persisted id 同时被两个 workspace runtime 恢复。same-workspace restore race 继续沿用 bridge 的 `restore_in_progress` / coalesce 行为。

live read-only session status routes 改为通过 owner runtime dispatch；`GET /session/:id/rewind/snapshots` 保持 primary-only，因为 rewind state 还不是 sessions-only loop 的一部分。workspace session list 增加 `/workspaces/:workspace/sessions` 复数别名，与 `/workspace/:id/sessions` 同形；路径参数先按 workspace id 精确匹配，再按 canonical cwd 匹配。primary workspace 保持 persisted/live merge，non-primary workspace 仍 live-only，拒绝 archive/organized/grouped 查询。

## 关键代码路径

- `packages/cli/src/serve/workspace-registry.ts`: `WorkspaceSessionOwnerIndex`、index-first owner resolution、stale cleanup、fallback scan 回填。
- `packages/acp-bridge/src/bridgeOptions.ts`: bridge `sessionLifecycle` callback seam。
- `packages/acp-bridge/src/bridge.ts`: spawn/load/resume/close/kill/channel exit 等生命周期事件接入 owner index。
- `packages/cli/src/serve/routes/session.ts`: trusted non-primary load/resume、`session_workspace_conflict`、read-only status routing。
- `packages/cli/src/serve/server/session-list.ts`: `/workspaces/:workspace/sessions` alias 与 non-primary live-only listing。
- `packages/cli/src/serve/run-qwen-serve.ts`、`server.ts`: internally-created bridge wiring owner index。
- `packages/cli/src/serve/workspace-registry.test.ts`、`multi-workspace-sessions.test.ts`、`server-default-bridge-wiring.test.ts`: owner index、restore conflict 和 route wiring 覆盖。
