## 解决的问题

multi-workspace daemon 已经能通过 capabilities 暴露多个 workspace，并能按 workspace-qualified route 访问 secondary runtime，但 Web Shell 仍主要是 primary session list：用户看不到完整 workspace tree，也不能在运行中的 daemon 里注册新的 workspace。若要添加 workspace，只能重启 daemon 或依赖外部启动参数，体验和 daemon 的动态管理目标不一致。

## 最终实现

该 PR 把 Web Shell sidebar 改成 workspace management surface。多 workspace capability 存在时，左侧栏按 workspace 渲染 `WorkspaceSection`，每个 section 显示 workspace cwd、primary badge、信任状态和该 workspace 下的 sessions；trusted workspace 可展开/折叠、刷新、创建/查看 session，untrusted workspace 保持可见但禁用操作。单 workspace daemon 继续保持简单 session list，并只补一个 add-workspace 入口。

Web Shell 新增 `AddWorkspaceDialog`，通过 SDK `DaemonClient.addWorkspace(cwd)` 调用 daemon。workspace provider 的 `actions.addWorkspace` 在成功后调用 `refreshCapabilities({ force:true })`，绕过原先 cached capabilities promise，让新增 workspace 立即出现在 UI；session preparation 和 prompt/session create 路径可以传一次性 `workspaceCwd`，每个 workspace section 用 `client.workspaceByCwd(cwd).listSessions(...)` 拉取自己的 session list。

服务端新增 strict mutation route `POST /workspaces`。路由要求 cwd 是非空绝对路径且长度受限，使用 `realpathSync.native` canonicalize，要求目标存在且是目录。它拒绝已注册 workspace、正在注册的重复 cwd、parent/child nested workspace，并避免把未 canonicalize 的用户输入泄露到错误消息。一个 in-flight Set 覆盖并发重复/嵌套注册 race；总注册数上限为 25。

runtime 创建复用 `run-qwen-serve` 的 workspace runtime 装配路径：加载 settings，构建 workspace service、bridge、fsFactory、client-MCP sender registry、env source，并注册进可变 `WorkspaceRegistry`。settings/env reload 失败时保留旧 runtime env，不把 half-built runtime 注入 registry。

- `packages/cli/src/serve/run-qwen-serve.ts`: dynamic workspace runtime creation、settings/env reload integration、registry add。
- `packages/cli/src/serve/workspace-registry.ts`: mutable `add(runtime)` 与 workspace lookup。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`: `addWorkspace()` 与 workspace client refresh support。
- `packages/web-shell/client/App.tsx`、`components/sidebar/WorkspaceSection.tsx`、`components/dialogs/AddWorkspaceDialog.tsx`: workspace sidebar、add dialog、per-workspace session list。
- `packages/webui/src/daemon/workspace/actions.ts`: `addWorkspace` action 和 capabilities refresh。
