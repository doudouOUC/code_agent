## 解决的问题

ACP 客户端会按 workspace boundary 拒绝 `/tmp/datastudio_cli_extract/...` 等本地生成文件，即使 daemon 进程本身可以读取。#6021 已经允许 ACP 在特定 workspace-boundary 拒绝后 fallback 到受管本地根，但默认 roots 没覆盖 POSIX `/tmp`，导致这类临时日志/产物仍读不到。

## 最终实现

`packages/cli/src/acp-integration/acpAgent.ts` 新增 ACP-only local read roots。POSIX 平台默认把 `/tmp` 追加到 fallback roots；Windows 不默认加入 `/tmp`。新增 `QWEN_ACP_LOCAL_READ_ROOTS` 作为 append-only escape hatch，只接受 `path.delimiter` 分隔的绝对路径，空项和相对路径被忽略。

该变更只影响 ACP local fallback roots，不改变普通 `read_file` 的默认权限策略。`packages/core/src/tools/read-file.ts` 保持外部 `/tmp` 读取默认仍 ask；ACP fallback 仍依赖 `AcpFileSystemService` 的 realpath/subpath 校验，只有在客户端 readTextFile 因 workspace boundary 失败且路径命中 fallback root 时才走 daemon 本地读。

## 关键代码路径

- `packages/cli/src/acp-integration/acpAgent.ts`: `defaultAcpOnlyLocalReadRoots()`、`parseAcpLocalReadRootsEnv()` 和 ACP FileSystemService 装配。
- `packages/cli/src/acp-integration/acpAgent.test.ts`: 默认 `/tmp` root、env 追加绝对路径、相对路径过滤和 session-specific fallback roots。
- `packages/core/src/tools/read-file.ts`: 普通 read_file 默认权限继续要求外部路径确认。
