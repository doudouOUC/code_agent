## 解决的问题

#6558 让 trusted non-primary workspace 可以看到 active recent persisted sessions，但 archived、organized 和 group 视图仍由 phase guard 拒绝。这样 secondary workspace 虽然能 load/resume 历史 session，却不能用与 primary 一致的 archive list、organized view 或 group filter 管理这些历史记录。

## 最终实现

该 PR 移除 non-primary workspace session list 的 archived/organized phase limitation。trusted non-primary route 现在用 selected runtime 的 workspace cwd 初始化 `SessionService` 和 session organization store，分别读取该 workspace 的 `chats/`、`chats/archive/` 与 `session-organization.v1.json`。

`archiveState=archived` 返回该 workspace archived sessions；`view=organized` 返回 group/pin 排序视图；`view=organized&archiveState=archived` 只读 archive 目录，不再混入 live summaries。`group` filter 仍只允许在 organized view 下使用，不带 organized view 时返回 `invalid_session_group_filter`；未知 group id 对齐 primary behavior 返回 `404 group_not_found`，不再用旧的 phase limitation 400。

untrusted workspace 仍按原策略拒绝；primary 和 legacy route 行为不变。

## 关键代码路径

- `packages/cli/src/serve/routes/session.ts`: non-primary session list 分支支持 archived/organized/group filter，改用 selected workspace storage。
- `packages/cli/src/serve/multi-workspace-sessions.test.ts`: trusted secondary archived/organized/group 视图覆盖，unknown group 与 invalid filter 覆盖。
- `docs/developers/qwen-serve-protocol.md`: multi-workspace session list 协议边界同步。

## 验证方式

- `cd packages/cli && npx vitest run src/serve/multi-workspace-sessions.test.ts -t "non-primary"`
- 手工验证：在 secondary workspace 创建 active、archived、pinned/grouped sessions 后，请求 `/workspaces/:workspace/sessions?archiveState=archived`、`?view=organized`、`?view=organized&group=<id>`，确认结果只来自对应 workspace。
