## 解决的问题

旧 `workspace_extensions` 主要是 primary `/workspace/extensions` 诊断面，无法清楚表达 multi-workspace daemon 下的两个层次：extension artifact 是用户级安装物，workspace 只应该持有启用/禁用/继承策略。若把安装、更新、卸载和 workspace activation 混在同一 workspace route 中，容易产生跨 workspace artifact ownership、并发安装/更新冲突、V1 downgrade 投影和 runtime refresh 不一致的问题。

## 当前实现方案

当前 open diff 新增 additive capability `extension_management_v2`，并明确不采用早期 workspace-qualified extensions 方案名。legacy `workspace_extensions` 和 `/workspace/extensions/*` 保持 primary compatibility adapter；V2 才是新管理面。

V2 资源模型拆成两层：安装 artifact 是 user-level，落在 `QWEN_HOME/extensions`；activation 是 policy，按 exact workspace override（enabled/disabled）、内部 exact inherit mask、V1 path rule 和 global default 计算 effective state。`ExtensionStore` 成为唯一 writer，负责最终 extension dirs、V2 activation state、V1 import/projection/downgrade，以及 `~/.qwen/extension-store/` 下的 `state.json`、`state.previous.json`、staging、trash、journal 和 recovery。atomic state write 与 journal recovery 确保 crash 后能恢复到可解释状态。

全局 route 处理 artifact 操作：`GET /extensions`、`POST /extensions/install`、`POST /extensions/check-updates`、`POST /extensions/:extensionId/update`、`DELETE /extensions/:extensionId`、`PUT /extensions/:extensionId/activation`、`GET /extensions/operations/:operationId`。workspace projection route 只处理 selected workspace 的 effective view、activation override 和 runtime refresh：`GET /workspaces/:workspace/extensions`、`PUT /workspaces/:workspace/extensions/:extensionId/activation`、`DELETE /workspaces/:workspace/extensions/:extensionId/activation`、`POST /workspaces/:workspace/extensions/refresh`。workspace route 不拥有 artifact mutation；read 可面向 untrusted workspace，activation/refresh/workspace-scoped install 仍要求 trusted。

慢操作返回 `202 Accepted` + `operationId`/`Location`/`Retry-After`。operation history 只在 daemon 本地保留，terminal records capped 100，daemon 重启后可消失；权威状态以 store generation 为准。daemon 最多接受 10 个 unfinished ops；preparation queue 并发 2，commit queue 单并发。prep deadline 从拿到 slot 后才开始，并把 AbortSignal 传到网络下载；commit 不可取消。prepared update 携带目标 artifact generation，stale same-artifact update 返回 `extension_conflict`，无关 extension operation 可 rebase。partial refresh/reload cleanup 以 `succeeded_with_warnings` 表达，secrets 在 response/log 中脱敏。

SDK 新增 global methods：`extensionCatalog()`、`installUserExtension()`、`checkUserExtensionUpdates()`、`updateUserExtension()`、`uninstallUserExtension()`、`setExtensionDefaultActivation()`、`extensionOperation()`、`waitForExtensionOperation()`；`WorkspaceDaemonClient` 新增 `workspaceExtensions()`、`setExtensionActivation()`、`clearExtensionActivation()`、`refreshExtensionRuntime()`。`waitForExtensionOperation()` 超时只停止客户端等待，不取消 daemon 已接受的后台操作。

## 关键代码路径

- `packages/core/src/extension/extension-store.ts`: V2 store、state/journal/recovery、activation policy、V1 projection。
- `packages/core/src/extension/archive-safety.ts`: 安装包 link/symlink 安全校验。
- `packages/core/src/extension/extensionManager.ts`: extension artifact/install/update/delete/runtime refresh integration。
- `packages/core/src/extension/github.ts`、`npm.ts`: AbortSignal-aware download/update integration。
- `packages/cli/src/config/extension-file-watcher.ts`: extension file watcher integration。
- `packages/cli/src/serve/routes/workspace-extensions.ts`、`workspace-extensions-controller.ts`: global `/extensions/*` operations 与 workspace projection routes。
- `packages/cli/src/serve/extension-operation-scheduler.ts`: FIFO preparation/commit queue primitive；controller 负责 operation admission、history、warnings 和 refresh。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`、`types.ts`: extension management v2 SDK helpers and types。
- `packages/cli/src/commands/extensions/install.ts`、`link.ts`: CLI install/link 对 V2 activation commit 和 warning handling 的适配。
- `packages/cli/src/ui/commands/extensionsCommand.ts` 与 `packages/cli/src/ui/components/extensions/`: TUI extension install/source/actions UI 对后台 operation 的适配。
