## 解决的问题

#6482 把 live `/load` 的 replay snapshot 改成有界内存窗口，解决了 daemon resident memory 增长问题。但客户端仍需要查看或重建完整 active persisted transcript，尤其是长会话的历史。如果继续让 `/load` 返回全量历史，会重新引入一次性大数组、EventBus seed 和 live replay window 膨胀问题。

## 最终实现

该 PR 增加 cursor-paged `GET /session/:id/transcript`。第一页由 `SessionTranscriptReader` 冻结 active JSONL 的 snapshot size 和文件身份，扫描该前缀建立轻量索引，只按 active `ChatRecord` parent chain 分页，默认 limit 100、最大 500。后续页通过 opaque base64url cursor 继续；cursor 包含 session id、snapshot size、file identity、position、leafUuid 和 replay state，并用 workspace project 目录下的持久 HMAC key 签名。key 按路径缓存到 daemon 生命周期内，缺失会生成，长度/内容损坏会替换；cursor 防篡改并绑定当前 workspace/file snapshot，但不承诺作为长期 bookmark 或跨文件替换继续使用。

路由层不直接把 records 暴露给客户端，而是通过 ACP child 的只读 `qwen/status/session/transcript` status method，把每页 records 转为 id-less `session_update` replay frames。该调用不 attach client、不 seed EventBus、不创建 live session、不改变 live replay window，也不返回 `lastEventId`。跨页 replay 状态会携带 tool-call continuation 和 cumulative usage，保证分页 replay 投影连续。

错误边界按 transcript 读取语义独立建模：invalid limit/cursor 返回 400；active transcript 不存在返回 404；archive/unarchive 冲突沿用 session archive coordinator；文件被删除、截断、替换或移到 archive 后，cursor 页返回 409，客户端应从第一页重开。索引 cache 是 bounded TTL/LRU，snapshot 超过 256 MiB 会在建索引前返回结构化 too-large 错误；bridge 侧读取有 60s backstop，ACP status method 会保留 replay 错误时的 `hasMore` 和 `nextCursor`，避免客户端误判分页完成。line scanner 使用 chunk list 管理 pending buffer，避免超大单行记录造成 quadratic copy。

SDK TS 新增 `DaemonClient.getSessionTranscriptPage(sessionId, { cursor, limit })`，capability tag 为 `session_transcript`。`history_truncated` marker 增加 `fullTranscriptAvailable`，让客户端区分“需要完整 transcript 可分页获取”与普通截断提示；browser-safe bundle size guard 随新增 API 调整。文档同步明确 `/load` 是 bounded live replay，完整 active persisted transcript 走 `/transcript` pages。

## 关键代码路径

- `packages/core/src/services/session-transcript-reader.ts`: frozen snapshot、active chain pagination、workspace project 持久 HMAC cursor key、index cache、large-line scan、256 MiB snapshot guard。
- `packages/cli/src/acp-integration/acpAgent.ts`: `qwen/status/session/transcript` read-only status method，把 records 转 replay frames。
- `packages/cli/src/acp-integration/session/HistoryReplayer.ts`: 支持 transcript page replay continuation。
- `packages/cli/src/serve/routes/session.ts`: `GET /session/:id/transcript` REST route、limit/cursor validation、archive/loadability guard。
- `packages/cli/src/serve/server/error-response.ts`: transcript-specific error mapping。
- `packages/acp-bridge/src/bridge.ts`、`bridgeTypes.ts`: `getSessionTranscriptPage` bridge seam。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`、`types.ts`: `getSessionTranscriptPage()` 与 page/cursor types。
- `packages/webui/src/daemon/session/DaemonSessionProvider.test.tsx`、`packages/vscode-ide-companion/src/ide-server.test.ts`: transcript event/status 兼容覆盖。
