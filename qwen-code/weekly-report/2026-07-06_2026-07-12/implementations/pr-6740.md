## 解决的问题

#6717 已经允许 untrusted secondary workspace 在信任前读取 persisted session catalog，但仍不能查看 active transcript 内容。若复用旧的 singular transcript route，会走 ACP-backed 读取链路，可能触发 attach、workspace settings load、live bridge 查询或 legacy persisted cursor-key 文件创建，不符合 untrusted secondary 的“只读、不执行、不初始化 runtime 状态”边界。

## 最终实现方案

新增 workspace-qualified persisted-only transcript route：`GET /workspaces/:workspace/session/:id/transcript`。路由先解析 selected workspace runtime，再从该 workspace 的 active persisted JSONL 冻结快照生成与旧 `GET /session/:id/transcript` 相同的 `DaemonSessionTranscriptPage` replay projection。该路径不 attach client、不启动 ACP、不查询 live bridge、不加载 workspace settings、不 discover project capabilities，也不创建旧 singular route 使用的 persisted cursor-key 文件。

cursor 签名从“项目目录持久 key”调整为 workspace 维度的 daemon-lifetime in-memory key：`SessionTranscriptCursorCodec` 支持注入 key provider，并为 workspace route 派生每个 workspace 的内存签名 key。这样 untrusted secondary 可以分页读取当前 daemon 生命周期内的 transcript，但 cursor 不能跨 daemon 重启长期复用。

能力层新增无条件 tag `workspace_persisted_transcript`，表示 daemon 支持该 workspace-qualified read-only pager。tag 无条件广告，但每次请求仍做 workspace selector 和 trust policy 校验：trusted primary/secondary 可读，registered untrusted secondary 可读 persisted-only transcript，untrusted primary 仍拒绝。SDK `WorkspaceDaemonClient.getSessionTranscriptPage()` 强制走 native REST，即使 `DaemonClient` 装了 replaceable transport，也不会绕到 ACP transport。

该 PR 同时把 legacy history replay 模块机械重命名为 kebab-case，并抽出 `collectHistoryReplayUpdates()` / `replayTranscriptRecordPage()`，让 persisted JSONL 到 id-less replay frames 的转换可以在没有 full `Config` 的 workspace route 中复用。

## 关键代码路径

- `packages/cli/src/serve/routes/session.ts`: 新增 `/workspaces/:workspace/session/:id/transcript` route，接入 selected workspace、trust policy、telemetry 和 error response。
- `packages/core/src/services/session-transcript-reader.ts`: cursor codec 支持注入签名 key，读取 active JSONL frozen snapshot 并分页。
- `packages/cli/src/acp-integration/session/history-replay-page.ts`: 抽出无 full Config 依赖的 transcript record page replay 转换。
- `packages/cli/src/acp-integration/session/types.ts`: 区分 full session context 与 workspace persisted reader 可用的轻量 context。
- `packages/cli/src/serve/server/error-response.ts`: workspace transcript route 的结构化错误映射。
- `packages/cli/src/serve/server/telemetry.ts`: transcript read 归因到目标 workspace/session。
- `packages/sdk-typescript/src/daemon/DaemonClient.ts`: `WorkspaceDaemonClient.getSessionTranscriptPage()` 直接调用 native REST。
