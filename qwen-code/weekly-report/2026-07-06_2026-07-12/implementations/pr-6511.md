## 解决的问题

#6378 需要一个安全的 multi-workspace daemon 路径，但 Phase 2a 不能一次性放开全部 daemon surface。核心风险是 non-primary workspace 的 live session creation、prompt、SSE、permission 等流量误走 primary runtime，或者让文件、MCP、settings、voice、channel-worker 等尚未迁移的 API 半启用。

## 最终实现

该 PR 打开 sessions-only 的 multi-workspace live loop。多个 distinct explicit `--workspace` 现在可以在 boot 时注册为 runtimes，第一个 workspace 保持 primary，旧的 workspace-less API 和 primary-only surface 仍只使用 primary runtime。重复、嵌套 workspace 输入继续 fail closed；`resolveWorkspaceInputs()` 保留输入顺序，用于 runtime 注册。

session 创建路由先用请求 `cwd` 精确匹配 `WorkspaceRegistry`。省略 `cwd` 回落 primary；unknown canonical cwd 返回 `400 workspace_mismatch`；registered 但 untrusted 的 non-primary workspace 返回 `403 untrusted_workspace`。成功创建时只触碰目标 runtime 的 bridge。live session 路由用 `WorkspaceRegistry.resolveLiveSessionOwner(sessionId)` 找 owner runtime，覆盖 events、prompt、cancel、session-scoped permission vote、heartbeat、detach、pending prompts、close、basic status 和 non-primary live-only session listing；ambiguous owner fail closed，miss 不会 fallback primary。

capabilities 和 status 只做 additive schema：当 registry 中 runtime 数量大于 1 时，`/capabilities.features` 发布 `multi_workspace_sessions`，并返回 `workspaces[]` 与 per-workspace/session limits；`workspaceCwd` 仍是 primary cwd，兼容旧客户端。SDK TS 只扩展 capability/status 类型与 docs，不增加 workspace-specific SDK client。persisted storage、file/MCP/settings/voice/channel-worker、ACP、export/archive/delete、branch/fork/cd/rewind/shell/model/language 等仍保持 primary-only 或后续阶段处理。

## 关键代码路径

- `packages/cli/src/serve/workspace-inputs.ts`: repeatable `--workspace` 解析、duplicate/nested guard、multi-workspace input order。
- `packages/cli/src/serve/run-qwen-serve.ts`: 多 runtime boot、primary runtime 保留、workspace registry 注入和 cleanup bridge 列表。
- `packages/cli/src/serve/workspace-registry.ts`: workspace id/cwd lookup、primary fallback、live owner scan。
- `packages/cli/src/serve/routes/session-runtime.ts`: live session owner resolution 的共享 fail-closed helper。
- `packages/cli/src/serve/routes/session.ts`、`routes/sse-events.ts`、`routes/permission.ts`: session 创建与 live session 路由按 runtime dispatch。
- `packages/cli/src/serve/server/session-list.ts`: non-primary live-only workspace session listing。
- `packages/cli/src/serve/routes/capabilities.ts`、`daemon-status.ts`: `multi_workspace_sessions`、`workspaces[]` 和 additive limits。
- `packages/sdk-typescript/src/daemon/types.ts`、`DaemonClient.ts`: additive capability/status type support。
