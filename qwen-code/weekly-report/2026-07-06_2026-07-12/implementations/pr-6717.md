## 解决的问题

multi-workspace daemon 可以注册 untrusted secondary workspace，但 #6631 之前的 session list/group route 对 untrusted secondary 基本拒绝。用户在决定是否信任一个 secondary repository 前，看不到它已有的 persisted session 名称、archive/organized/group 信息，只能在完全信任后再查看历史。这不利于判断工作区上下文，同时也不能为了可见性而启动 bridge、ACP 或修改该 workspace 的存储。

## 最终实现方案

该 PR 给 registered untrusted secondary workspace 开一个窄的只读目录面：只允许读取 persisted session catalog 与 session-group catalog，不查询 live bridge state、不 spawn ACP、不 load settings、不修复 storage，也不开放任何 mutation。primary workspace 仍维持原有 untrusted 行为；trusted workspace 路径不变。

`routes/session.ts` 新增 `isReadOnlyWorkspaceCatalog(runtime)` 判断 `!runtime.primary && !runtime.trusted`，并把 session list 与 session groups 的 runtime 解析从强制 trusted 改成 catalog route 专用解析。plural `/workspaces/:workspace/sessions` 和 `/workspaces/:workspace/session-groups` 对 untrusted secondary 返回 persisted-only 结果；legacy singular route 对 untrusted primary 继续保持原行为。所有 untrusted catalog read 包在 `runWithoutDebugLogSession()` 中，避免 file-backed debug logger 因为只读枚举而创建或写入 workspace 内部日志。

`server/session-list.ts` 给 `listWorkspaceSessionsForResponse()` 增加 `readOptions.mergeLive`。read-only catalog 调用时传 `mergeLive:false`，因此 active recent、organized、parent-session、archived、cursor/page-size 等查询都只来自 `SessionService(workspaceCwd)` 的 persisted JSONL 与 organization sidecar，不会把 `bridge.listWorkspaceSessions()` 的 live summary 混进来，也不会触碰 runtime-backed session state。trusted path 默认 `mergeLive` 仍为 true，保持原先 persisted/live merge 体验。

Web Shell sidebar 对 untrusted secondary 改为可展开但不可选择：只加载一次 session list，不开启 10s polling；行渲染为非交互 `role="note"` 的 read-only row，只显示 displayName 或 session id 前 8 位和创建时间，并附上 trust-to-open 提示。untrusted primary 仍禁用，不暴露 workspace 切换。

## 关键代码路径

- `packages/cli/src/serve/routes/session.ts`: catalog route runtime 解析、untrusted secondary read-only gate、debug log suppression。
- `packages/cli/src/serve/server/session-list.ts`: `mergeLive:false` persisted-only listing。
- `packages/core/src/utils/debugLogger.ts`: `runWithoutDebugLogSession()`，让 catalog read 不产生 debug log side effect。
- `packages/web-shell/client/components/sidebar/WorkspaceSection.tsx`: untrusted secondary read-only session rows、no polling、no workspace selection。
- `packages/cli/src/serve/multi-workspace-sessions.test.ts`: archived、organized、group、parent/cursor、untrusted trust gate 与 no bridge spawn 覆盖。
