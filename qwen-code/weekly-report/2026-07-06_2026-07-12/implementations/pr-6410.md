## 解决的问题

#6378 的 multi-workspace sessions 不能直接打开 route dispatch。打开前必须先把 CLI parser、workspace input guard、runtime registry、server assembly 注入边界和 live session owner resolution 固定下来。否则重复 `--workspace` 会被 yargs/fast path 的 parser 差异吞掉，后续多 workspace 路由也缺少 fail-closed 的 owner 查找边界。

## 最终实现

`packages/cli/src/commands/serve.ts` 把 `--workspace` 改为 `array: true`，确保单值和多值都以数组保留。fast path 遇到重复 `--workspace` 时返回 `{kind:'fallback'}`，交给完整 parser，避免轻量 parser 在 future multi-workspace 场景下产生不同语义。

`packages/cli/src/serve/workspace-inputs.ts` 集中处理 workspace 输入：未传值回退 `process.cwd()`；空数组报 `MissingWorkspaceInputError`；重复 canonical cwd 报 `DuplicateWorkspaceInputError`；父子嵌套路径报 `NestedWorkspaceInputError`；互不嵌套的多个路径报 `MultipleWorkspaceInputError`。这些错误都保留 "Multi-workspace serve is not enabled; pass one --workspace." 语义，明确当前 public 行为仍是单 workspace。

`packages/cli/src/serve/workspace-registry.ts` 扩展 `WorkspaceRuntime`，加入 `workspaceId`、`workspaceCwd`、`primary`、`trusted` 和 env metadata。`createWorkspaceRegistry()` 要求 exactly one primary，并为 cwd/id 建 map；`resolveWorkspaceCwd(undefined)` 返回 primary；`resolveLiveSessionOwner(sessionId)` 扫描 runtime bridge summary，0 个返回 not_found，1 个返回 found，多个返回 ambiguous，避免未来重复 session id 被静默路由到 primary。

`createServeApp()` 可接收注入的 `workspaceRegistry`。注入时 primary runtime 成为 bridge、workspace service、route fsFactory 和 client-MCP sender registry 的唯一来源；如果调用方同时传了不一致的 legacy deps，会直接 throw split-brain 冲突。未注入时仍创建 single-runtime registry，保留 `app.locals.boundWorkspace` / `app.locals.fsFactory` / `/capabilities.workspaceCwd` 的 primary-only 兼容行为。

## 关键代码路径

- `packages/cli/src/commands/serve.ts`: `--workspace` repeatable parser 边界。
- `packages/cli/src/serve/fast-path.ts`: 重复 `--workspace` 回退完整 parser。
- `packages/cli/src/serve/workspace-inputs.ts`: duplicate / nested / distinct multi-workspace fail-closed 逻辑。
- `packages/cli/src/serve/workspace-registry.ts`: runtime metadata、cwd/id lookup、primary fallback、live owner resolution。
- `packages/cli/src/serve/server.ts`: registry injection、依赖冲突检测、legacy locals 兼容。
- `.qwen/design/daemon-multi-workspace-phase2a-sessions.md`: Phase 2a foundation contract 与 route classification。
