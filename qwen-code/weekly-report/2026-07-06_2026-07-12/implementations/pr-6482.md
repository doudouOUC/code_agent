## 解决的问题

live daemon session 会把 replay history 保存在内存里，供后来的 `POST /session/:id/load` 客户端恢复 UI。长时间运行的 live session、以及 response-mode restore 一次性 seed 的历史 replay，都可能让这个内存窗口持续增长。旧语义还容易让客户端误以为 `/load` 会永远返回完整 transcript，导致 ring eviction 后的恢复边界不清楚。

## 最终实现

该 PR 把 live in-memory replay 明确改成有界 snapshot window。`TurnBoundaryCompactionEngine` 不再只维护无界 flat replay，而是把 completed live turn 和 restore/bulk replay 以 segment 形式保留，并按序列化字节上限裁剪旧 segment。默认上限是 4 MiB，硬上限是 256 MiB；单个最新 segment 即使超过 cap 也会保留，避免 snapshot 为空。

当旧 replay 被丢弃时，`snapshot()` 会在 `compactedReplay[0]` 前置一个无 id 的 `history_truncated` 事件，payload 包含 `reason:'replay_window_exceeded'`、`truncatedEvents`、`retainedEvents`、`maxBytes`、可选 `truncatedTurns` 和 `fullTranscriptAvailable`。该 marker 是状态提示，不消耗 EventBus id，也不会写回 replay window。`QWEN_SERVE_DEBUG` 打开时，replay eviction 会输出低敏 debug 计数，便于定位窗口被裁剪的原因。

配置链路新增 `--compacted-replay-max-bytes`，并贯通 yargs、serve fast path、`ServeOptions`、server/bridge wiring、`BridgeOptions.compactedReplayMaxBytes` 与 `/daemon/status.limits.compactedReplayMaxBytes`。非法值在 listen 前 fail closed。SDK TS 的 known-event 校验、UI normalizer、transcript reducer 和 WebUI `DaemonSessionProvider` 都识别 `history_truncated`：合法 payload 渲染为 transcript status，畸形 payload 走 debug fallback，replay injection 不会设置 `awaitingResync`，也不会触发二次 reload。

## 关键代码路径

- `packages/acp-bridge/src/compactionEngine.ts`: replay segment、byte cap、oldest segment eviction、`history_truncated` marker。
- `packages/acp-bridge/src/replayWindowLimits.ts`: 默认值、硬上限和 normalize helper。
- `packages/acp-bridge/src/eventBus.ts`: replay seed/snapshot 与 transient marker 过滤。
- `packages/acp-bridge/src/bridge.ts`: bridge 创建时注入 `compactedReplayMaxBytes`，load/restore snapshot 使用有界窗口。
- `packages/cli/src/commands/serve.ts`、`serve/fast-path.ts`、`serve/run-qwen-serve.ts`: CLI option、fast-path validation 和 boot-time validation。
- `packages/cli/src/serve/daemon-status.ts`: daemon status 暴露 resolved replay byte limit。
- `packages/sdk-typescript/src/daemon/events.ts`、`daemon/ui/normalizer.ts`: `history_truncated` typed event 和 UI status projection。
- `packages/webui/src/daemon/session/DaemonSessionProvider.tsx`: replay injection 中把 truncation marker 当作 status，而非 resync trigger。
