## 解决的问题

daemon settings reload 事件会广播到活动 session，但 Web Shell / SDK UI normalizer 之前不认识 `settings_reloaded`。结果是 reload payload 被当成未知 daemon debug event 渲染进对话 transcript，用户能看到 `settings_reloaded (unrecognized daemon event)` 这类运维噪声；同时开发者也缺少一条只含诊断字段的 reload 日志。

## 最终实现

`packages/sdk-typescript/src/daemon/ui/normalizer.ts` 为 `settings_reloaded` 增加专门分支。payload 是 object 时，事件被转换成 `workspace.settings.changed`，`key` 固定为 `settings_reloaded`，`scope` 为 `workspace`，原始 reload metadata 放在 `value` 中；payload 不是 object 时才回退到 malformed debug。

`packages/webui/src/daemon/session/DaemonSessionProvider.tsx` 在 normalize/filter 前调用 `logSettingsReloadEvent()`。日志只写 `console.debug('[DaemonSessionProvider] settings reloaded:', data)`，并通过 `getSettingsReloadLogData()` 过滤字段：`eventId`、`env.updatedKeys`、`env.removedKeys`、`changedKeys`、`childReloaded`、`sessionsRefreshed`、`sessionsSkipped`、`childError`。数组只保留 string item，非 object payload 只记录 `payload: 'non-object'`。

由于 normalizer 已把 `settings_reloaded` 归为 workspace settings signal，`filterDaemonUiEventsForTranscript()` 不再收到未知 debug event，transcript 不再插入 reload block；workspace event signal 的 `settingsVersion` 递增仍能驱动 UI 刷新。

## 关键代码路径

- `packages/sdk-typescript/src/daemon/ui/normalizer.ts`: `normalizeSettingsReloaded()` 把 daemon event 转成 `workspace.settings.changed`。
- `packages/sdk-typescript/test/unit/daemonUi.test.ts`: 覆盖 `settings_reloaded` 不再归一化成 debug。
- `packages/webui/src/daemon/session/DaemonSessionProvider.tsx`: reload event 诊断日志和安全字段筛选。
- `packages/webui/src/daemon/session/DaemonSessionProvider.test.tsx`: 覆盖 settings signal 增量、transcript 无 debug block、console debug 字段。
